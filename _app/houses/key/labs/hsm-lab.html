Envelope Encryption Pattern:
1. Call GenerateDataKey(KeyId) ‚Üí Returns plaintext + wrapped DEK
2. Use plaintext DEK to encrypt file locally (AES-256-GCM)
3. Store: encrypted file + wrapped DEK (+ Key ID for reference)
4. DELETE plaintext DEK from memory
5. To decrypt: Call KMS.Decrypt(wrapped DEK) ‚Üí Get plaintext DEK ‚Üí Decrypt file

Master key NEVER leaves KMS. Only DEKs are wrapped/unwrapped.

Remediate Hardcoded Key Vulnerability

Scenario

A security scan discovered hardcoded encryption keys in your application code. The code was committed to GitHub 6 months ago. Design a remediation plan.

Vulnerable Code Found:

// File: src/utils/encryption.js
const ENCRYPTION_KEY = 'a3f8b2c1d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1';

function encryptData(plaintext) {
    return CryptoJS.AES.encrypt(plaintext, ENCRYPTION_KEY).toString();
}

function decryptData(ciphertext) {
    return CryptoJS.AES.decrypt(ciphertext, ENCRYPTION_KEY).toString(CryptoJS.enc.Utf8);
}
üí° Need a hint?
Remediation Steps:
1. Assess Impact: All data encrypted in past 6 months is potentially compromised
2. Emergency Rotation: Generate new key in KMS, re-encrypt all affected data
3. Git History: Use BFG Repo-Cleaner to purge key from git. Force push. Notify contributors to re-clone.
4. GitHub Scan: Check for forks, pull requests that may contain the key
5. New Architecture: Use AWS Secrets Manager or KMS. Never store keys in code.
6. Notification: If PII/payment data affected, likely need customer + regulatory notification (GDPR, state laws)

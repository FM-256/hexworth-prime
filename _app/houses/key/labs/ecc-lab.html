Exercise 4: Verify ECDSA Signature
Verify the authenticity of a signed message
Intermediate
Objective
Verify that a signature is valid using the corresponding public key.
OpenSSL Terminal
# Verify the signature
$ openssl dgst -sha256 -verify ecc-public.pem -signature signature.der message.txt
Verified OK
# Test with modified message (should fail)
$ echo "Modified message" > tampered.txt
$ openssl dgst -sha256 -verify ecc-public.pem -signature signature.der tampered.txt
Verification Failure
# Even a single bit change will cause verification to fail
# This demonstrates the integrity protection of digital signatures
Verification Process
  1. Hash the message with SHA-256
  2. Use public key and signature (r, s) to compute verification point
  3. Check if computed value matches r from signature
  4. If match: signature is valid, message is authentic
  5. If no match: signature invalid, message may be tampered
Hint
Signature verification is mathematically guaranteed:
‚Ä¢ Anyone with the public key can verify
‚Ä¢ Only the private key holder can create valid signatures
‚Ä¢ Any change to the message invalidates the signature

Exercise 5: ECDH Key Agreement
Simulate a key exchange between two parties
Advanced
Objective
Perform an ECDH key exchange where Alice and Bob derive a shared secret.
OpenSSL Terminal
# Alice generates her key pair
$ openssl ecparam -name prime256v1 -genkey -noout -out alice-private.pem
$ openssl ec -in alice-private.pem -pubout -out alice-public.pem
# Bob generates his key pair
$ openssl ecparam -name prime256v1 -genkey -noout -out bob-private.pem
$ openssl ec -in bob-private.pem -pubout -out bob-public.pem
# Alice computes shared secret using her private key and Bob's public key
$ openssl pkeyutl -derive -inkey alice-private.pem -peerkey bob-public.pem -out alice-secret.bin
# Bob computes shared secret using his private key and Alice's public key
$ openssl pkeyutl -derive -inkey bob-private.pem -peerkey alice-public.pem -out bob-secret.bin
# Verify both secrets match
$ diff alice-secret.bin bob-secret.bin
(no output = files are identical)
ECDH Magic
Why it works:

Alice computes: S = a √ó (b √ó G) = (a √ó b) √ó G
Bob computes: S = b √ó (a √ó G) = (b √ó a) √ó G

Since multiplication is commutative: (a √ó b) √ó G = (b √ó a) √ó G

Both arrive at the same point S, which can be used to derive encryption keys.
Hint
The pkeyutl command is used for public key operations:
‚Ä¢ -derive: Perform key derivation (ECDH)
‚Ä¢ -inkey: Your private key
‚Ä¢ -peerkey: Other party's public key
The output is the shared secret (x-coordinate of shared point).

Exercise 6: Key Format Conversion
Convert ECC keys between PEM and DER formats
Intermediate
Objective
Learn to convert keys between different encoding formats used in various applications.
OpenSSL Terminal
# Convert private key from PEM to DER
$ openssl ec -in ecc-private.pem -outform DER -out ecc-private.der
# Convert public key from PEM to DER
$ openssl ec -pubin -in ecc-public.pem -outform DER -out ecc-public.der
# Convert back to PEM
$ openssl ec -inform DER -in ecc-private.der -out restored.pem
# View file sizes
$ ls -lh ecc-*
-rw------- 1 user user 227 Dec 20 ecc-private.pem (Base64-encoded)
-rw------- 1 user user 121 Dec 20 ecc-private.der (Binary)
-rw------- 1 user user 178 Dec 20 ecc-public.pem
-rw------- 1 user user 91 Dec 20 ecc-public.der
Format Differences
PEM (Privacy Enhanced Mail):
‚Ä¢ Base64-encoded text
‚Ä¢ Wrapped in BEGIN/END markers
‚Ä¢ Human-readable, email-safe
‚Ä¢ ~33% larger than DER

DER (Distinguished Encoding Rules):
‚Ä¢ Binary format
‚Ä¢ More compact
‚Ä¢ Used in certificates, PKCS formats
‚Ä¢ Not human-readable
Hint
Format selection depends on use case:
‚Ä¢ PEM: Web servers, configuration files, version control
‚Ä¢ DER: Java keystores, Windows certificates, embedded systems
Both formats contain the same cryptographic data.

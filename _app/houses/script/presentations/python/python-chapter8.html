Chapter 8: Object-Oriented Programming

What is OOP?

Object-Oriented Programming (OOP) is a way of organizing code around "objects" - bundles of data and behavior that model real-world things.

üéØ Key Idea

Think of a class as a blueprint and an object as a building made from that blueprint. You can create many buildings from one blueprint!

Creating a Class

Basic Class Structure
class Dog: """A simple Dog class.""" # Class attribute (shared by all instances) species = "Canis familiaris" # Constructor (initializer) def __init__(self, name, age): # Instance attributes (unique to each object) self.name = name self.age = age # Instance method def bark(self): return f"{self.name} says Woof!" # Another method def birthday(self): self.age += 1 return f"{self.name} is now {self.age}!"

Creating Objects

Instantiation
# Create instances (objects) buddy = Dog("Buddy", 3) max = Dog("Max", 5) # Access attributes print(buddy.name) # Buddy print(max.age) # 5 print(buddy.species) # Canis familiaris # Call methods print(buddy.bark()) # Buddy says Woof! print(max.birthday()) # Max is now 6!

Inheritance

Create a new class that inherits attributes and methods from a parent class.

Parent and Child Classes
# Parent class class Animal: def __init__(self, name): self.name = name def speak(self): raise NotImplementedError # Child classes class Dog(Animal): def speak(self): return f"{self.name} says Woof!" class Cat(Animal): def speak(self): return f"{self.name} says Meow!" # Polymorphism in action animals = [Dog("Buddy"), Cat("Whiskers")] for animal in animals: print(animal.speak()) # Buddy says Woof! # Whiskers says Meow!

Special Methods (Dunder Methods)

__init__

Constructor - runs when object is created.

def __init__(self, x): self.x = x
__str__

Human-readable string representation.

def __str__(self): return f"Point({self.x})"
__repr__

Developer-readable representation.

def __repr__(self): return f"Point(x={self.x})"
__len__

Called by len() function.

def __len__(self): return len(self.items)

Encapsulation: Public vs Private

Access Modifiers (Convention)
class BankAccount: def __init__(self, balance): self._balance = balance # Protected (by convention) self.__secret = "hidden" # Private (name mangling) def deposit(self, amount): if amount > 0: self._balance += amount def get_balance(self): return self._balance account = BankAccount(1000) account.deposit(500) print(account.get_balance()) # 1500

Real-World Example

User Management System
class User: def __init__(self, username, email): self.username = username self.email = email self._password = None def set_password(self, password): # Would hash in production! self._password = password def check_password(self, password): return self._password == password def __str__(self): return f"User({self.username})" # Create a user user = User("alice", "alice@example.com") user.set_password("secret123") print(user.check_password("secret123")) # True

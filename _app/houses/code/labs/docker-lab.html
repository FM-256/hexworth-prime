Pull a Docker Image
Pending
Download the official NGINX web server image from Docker Hub. This is the first step in working with Docker - getting images from the registry.
Command
docker pull nginx:latest
Verification Steps:
  • Open your terminal or command prompt
  • Run the command above
  • Wait for the download to complete
  • Verify with: docker images
Hint:

If you don't have Docker installed, you can practice the commands in our Docker Playground simulator instead!

2
Run Your First Container
Pending
Start an NGINX container in detached mode and map it to port 8080 on your host machine. This creates a running web server you can access in your browser.
Command
docker run -d -p 8080:80 --name my-nginx nginx:latest
Explanation
-d: Run in detached mode (background)
-p 8080:80: Map host port 8080 to container port 80
--name my-nginx: Give the container a friendly name
nginx:latest: The image to use
Verification Steps:
  • Run the command to start the container
  • Check it's running: docker ps
  • Open browser and visit: http://localhost:8080
  • You should see the NGINX welcome page!
3
Inspect and Manage Container
Pending
Learn to inspect container logs, stop the container, and then restart it. These are essential container management skills.
View Container Logs
docker logs my-nginx
Stop the Container
docker stop my-nginx
Start it Again
docker start my-nginx
Verification Steps:
  • View logs to see NGINX startup messages
  • Stop the container and verify: docker ps (should not show it)
  • Use docker ps -a to see stopped containers
  • Start it again and verify it's running

Build a Custom Docker Image
Pending
Create a simple custom image with a Dockerfile. This teaches you how to package your own applications.
Step 1: Create a Dockerfile
# Create a new directory mkdir my-docker-app && cd my-docker-app # Create Dockerfile (use your text editor) # Add the following content: FROM nginx:alpine COPY index.html /usr/share/nginx/html/index.html
Step 2: Create index.html
<!DOCTYPE html> <html> <head><title>My Docker App</title></head> <body> <h1>Hello from My Custom Docker Image!</h1> <p>This is running in a container I built myself.</p> </body> </html>
Step 3: Build the Image
docker build -t my-custom-app:v1 .
Step 4: Run Your Custom Container
docker run -d -p 8081:80 --name my-app my-custom-app:v1
Verification Steps:
  • Build completes successfully with no errors
  • Verify image exists: docker images
  • Run the container on port 8081
  • Visit http://localhost:8081 to see your custom page

Use Docker Compose
Pending
Create a multi-container application using Docker Compose. This simulates a real-world web app with a database.
Create docker-compose.yml
version: '3.8' services: web: image: nginx:alpine ports: - "8082:80" volumes: - ./html:/usr/share/nginx/html db: image: postgres:14-alpine environment: POSTGRES_PASSWORD: mysecretpassword POSTGRES_DB: myapp volumes: - db-data:/var/lib/postgresql/data volumes: db-data:
Start All Services
docker-compose up -d
View Running Services
docker-compose ps
Stop All Services
docker-compose down
Verification Steps:
  • Create the docker-compose.yml file
  • Run docker-compose up -d
  • Verify both services are running: docker-compose ps
  • Clean up: docker-compose down
Hint:

Docker Compose requires a separate installation in some cases. If you don't have it, you can still mark this complete after understanding the concept!

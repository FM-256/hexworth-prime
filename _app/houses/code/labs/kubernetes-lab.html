Create a Simple Pod
Pending
Create your first Kubernetes Pod using a YAML manifest. This Pod will run a single NGINX container.
nginx-pod.yaml
apiVersion: v1 kind: Pod metadata: name: nginx-pod labels: app: nginx spec: containers: - name: nginx image: nginx:latest ports: - containerPort: 80
Apply the Pod
kubectl apply -f nginx-pod.yaml
Verification Steps:
  • Create the YAML file with the content above
  • Run the kubectl apply command
  • Check pod status: kubectl get pods
  • Verify it's running: kubectl describe pod nginx-pod
Hint:

If you don't have a Kubernetes cluster, you can use Minikube locally or try our Kubernetes Simulator to practice the commands!

Create a Deployment
Pending
Create a Deployment that manages 3 replicas of an NGINX pod. Deployments provide self-healing and scaling capabilities.
nginx-deployment.yaml
apiVersion: apps/v1 kind: Deployment metadata: name: nginx-deployment labels: app: nginx spec: replicas: 3 selector: matchLabels: app: nginx template: metadata: labels: app: nginx spec: containers: - name: nginx image: nginx:1.24 ports: - containerPort: 80 resources: requests: memory: "64Mi" cpu: "100m" limits: memory: "128Mi" cpu: "200m"
Apply the Deployment
kubectl apply -f nginx-deployment.yaml
Verification Steps:
  • Apply the deployment YAML
  • Check deployment status: kubectl get deployments
  • Verify all 3 pods are running: kubectl get pods
  • Check rollout status: kubectl rollout status deployment/nginx-deployment

Expose with a Service
Pending
Create a Service to expose your NGINX deployment. Services provide stable networking and load balancing across pods.
nginx-service.yaml
apiVersion: v1 kind: Service metadata: name: nginx-service spec: selector: app: nginx type: LoadBalancer ports: - protocol: TCP port: 80 targetPort: 80 nodePort: 30080
Apply the Service
kubectl apply -f nginx-service.yaml
View the Service
kubectl get services kubectl describe service nginx-service
Verification Steps:
  • Apply the service YAML
  • Check service status: kubectl get svc
  • Verify endpoints are populated: kubectl get endpoints nginx-service
  • Test access (if using Minikube): minikube service nginx-service
Note:

LoadBalancer type requires a cloud provider or MetalLB. For local testing with Minikube, use NodePort instead or the minikube service command.

Scale the Deployment
Pending
Practice scaling your deployment up and down. This demonstrates Kubernetes' ability to dynamically adjust resources.
Scale Up to 5 Replicas
kubectl scale deployment nginx-deployment --replicas=5
Watch Pods Being Created
kubectl get pods -w
Scale Down to 2 Replicas
kubectl scale deployment nginx-deployment --replicas=2
Verification Steps:
  • Scale up to 5 replicas and verify: kubectl get deployment nginx-deployment
  • Confirm all 5 pods are running: kubectl get pods
  • Scale down to 2 replicas
  • Verify 2 pods remain: kubectl get pods
Pro Tip:

Use kubectl get pods -w (watch mode) to see pods being created or terminated in real-time. Press Ctrl+C to exit watch mode.

Perform a Rolling Update
Pending
Update your deployment to use a different NGINX version. Kubernetes will perform a rolling update with zero downtime.
Update Image Version
kubectl set image deployment/nginx-deployment nginx=nginx:1.25
Watch the Rollout
kubectl rollout status deployment/nginx-deployment
View Rollout History
kubectl rollout history deployment/nginx-deployment
Rollback if Needed
kubectl rollout undo deployment/nginx-deployment
Verification Steps:
  • Update the image to nginx:1.25
  • Watch the rollout progress complete successfully
  • Verify new version: kubectl describe deployment nginx-deployment
  • Optional: Test rollback and verify it returns to previous version
Production Tip:

Always test updates in a staging environment first. Use kubectl rollout pause to pause a rollout if issues are detected, investigate, then kubectl rollout resume or undo.

Essential Docker Commands

üîç docker pull
Download an image from Docker Hub
Example
docker pull nginx:latest docker pull ubuntu:22.04

Downloads a Docker image from a registry. If no tag is specified, Docker pulls the 'latest' tag by default.

üìã docker images
List all downloaded images
Example
docker images docker images -a

Shows all images on your system with their repository, tag, image ID, creation date, and size.

‚ñ∂Ô∏è docker run
Create and start a container from an image
Example
docker run -d -p 80:80 nginx docker run -it ubuntu bash docker run --name myapp -d node:18

The most important Docker command. -d runs in background (detached), -p maps ports, -it creates interactive terminal, --name gives it a friendly name.

üìä docker ps
List running containers
Example
docker ps docker ps -a

Shows running containers. Use -a flag to see all containers including stopped ones.

‚èπÔ∏è docker stop
Stop a running container gracefully
Example
docker stop container_name docker stop abc123

Sends SIGTERM signal, waits for graceful shutdown. Use container name or ID.

üîÑ docker start
Start a stopped container
Example
docker start container_name

Restarts a container that was previously stopped. The container retains its configuration and state.

üóëÔ∏è docker rm
Remove a stopped container
Example
docker rm container_name docker rm -f running_container

Deletes a container. Use -f to force remove a running container (not recommended).

üìù docker logs
View container output and logs
Example
docker logs container_name docker logs -f container_name

Shows stdout and stderr from a container. Use -f to follow logs in real-time (like tail -f).

üî® docker build
Build an image from a Dockerfile
Example
docker build -t myapp:1.0 . docker build -t username/app:latest .

Creates an image from a Dockerfile. -t tags the image with a name, . specifies build context (current directory).

Dockerfile Basics

A Dockerfile is a text file containing instructions to build a Docker image. Each instruction creates a new layer in the image.

# Start from a base image FROM node:18-alpine # Set working directory inside container WORKDIR /app # Copy package files COPY package*.json ./ # Install dependencies RUN npm install # Copy application source code COPY . . # Expose port for application EXPOSE 3000 # Command to run when container starts CMD ["npm", "start"]

Key Instructions:
FROM - Base image to build upon
WORKDIR - Sets working directory
COPY - Copies files from host to container
RUN - Executes commands during build
EXPOSE - Documents which ports to expose
CMD - Default command when container starts

Docker Compose Introduction

Docker Compose is a tool for defining and running multi-container applications. Instead of running complex docker run commands, you define everything in a YAML file.

# docker-compose.yml version: '3.8' services: web: build: . ports: - "3000:3000" environment: - NODE_ENV=production depends_on: - db db: image: postgres:14 environment: - POSTGRES_PASSWORD=secret volumes: - db-data:/var/lib/postgresql/data volumes: db-data:

Common Commands:
docker-compose up - Start all services
docker-compose up -d - Start in background
docker-compose down - Stop and remove containers
docker-compose logs - View logs from all services

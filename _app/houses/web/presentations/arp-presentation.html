Why ARP Exists - The Problem

The Layer 2/3 Mapping Challenge

Networks operate at multiple layers simultaneously:

Layer 3 - Network Layer

  • IP Addresses: Logical addresses
  • Example: 192.168.1.100
  • Used for routing across networks
  • Can change as device moves

Layer 2 - Data Link Layer

  • MAC Addresses: Physical addresses
  • Example: 00:1A:2B:3C:4D:5E
  • Used for local delivery
  • Burned into network card

The Problem

Applications know the destination IP address, but Ethernet frames need the destination MAC address for local delivery. How do we find the MAC address when we only know the IP?

The Solution

ARP (Address Resolution Protocol) dynamically maps IP addresses to MAC addresses on local networks.

What is ARP?

Definition

ARP is a network protocol used to discover the MAC address associated with a given IP address on a local network segment.

Key Characteristics

  • Layer: Operates between Layer 2 and Layer 3
  • EtherType: 0x0806 (in Ethernet frames)
  • RFC: RFC 826 (1982)
  • Scope: Works only on local network segment
  • Request Type: Broadcast (FF:FF:FF:FF:FF:FF)
  • Reply Type: Unicast (specific MAC address)
  • No Authentication: Trust-based protocol (security risk)

Important Note

ARP only works for IPv4. IPv6 uses NDP (Neighbor Discovery Protocol) instead.

ARP Cache / ARP Table

ARP Cache

IP Address MAC Address Type
192.168.1.1 00:11:22:33:44:55 Dynamic
192.168.1.20 AA:BB:CC:DD:EE:FF Dynamic
192.168.1.100 11:22:33:44:55:66 Static

ARP Cache Types

  • Dynamic Entries: Learned automatically, expire after timeout
  • Static Entries: Manually configured, never expire
  • Purpose: Avoid repeated ARP requests for same destination
  • Benefits: Reduces network traffic, faster communication

Why Aging Matters

  • Network Changes: Devices can change, IP reassignments
  • Memory Management: Prevent cache from growing indefinitely
  • Stale Entries: Remove entries for devices no longer present
  • Security: Limit impact of poisoned entries (though not reliable)

Gratuitous ARP (GARP)

Device

What is Gratuitous ARP?

An unsolicited ARP reply where a device announces its own IP-to-MAC mapping, even when no one asked.

Gratuitous ARP Purposes

  • IP Conflict Detection: Check if another device is using same IP
  • Cache Update: Update neighbors' ARP caches with new MAC
  • Failover/VRRP: Notify network of IP address takeover
  • Interface Changes: Announce new MAC after interface change

GARP Characteristics

Sender IP = Target IP (device announces itself)
Broadcast: Sent to FF:FF:FF:FF:FF:FF
Can be request or reply format

Slide 8

Proxy ARP

What is Proxy ARP?

Definition

A router responds to ARP requests on behalf of another device, typically on a different network segment.

How Proxy ARP Works

Step 1

PC sends ARP request for remote IP

Step 2

Router intercepts request

Step 3

Router replies with its own MAC

Step 4

PC sends packets to router

Use Cases

  • Misconfigured Hosts: Hosts with wrong subnet mask
  • Legacy Networks: Older networks without proper routing
  • Transparent Bridging: Make routing transparent to hosts

Caution

Proxy ARP is disabled by default on most modern routers. It can mask network configuration problems and is considered a legacy feature.

ARP in Different Scenarios

Local Network Communication

Same Subnet

Scenario: PC-A (192.168.1.10) ‚Üí PC-B (192.168.1.20)
ARP Target: 192.168.1.20 (destination IP)
Result: Direct communication using PC-B's MAC address

Remote Network Communication

Different Subnet

Scenario: PC-A (192.168.1.10) ‚Üí Server (10.0.0.5)
ARP Target: 192.168.1.1 (default gateway)
Result: Packet sent to router's MAC, router forwards to destination

Key Principle

Important Understanding

ARP always resolves the next-hop IP address, not necessarily the final destination!

  • If destination is local: ARP for destination IP
  • If destination is remote: ARP for default gateway IP

Slide 10

ARP Security Issues

Major Security Flaw

ARP has NO AUTHENTICATION mechanism. Any device can claim any IP/MAC mapping, and others will believe it!

Common ARP Attacks

ARP Spoofing / Poisoning

  • Attacker sends fake ARP replies
  • Victims update cache with false mappings
  • Traffic redirected to attacker
  • Enables Man-in-the-Middle attacks

ARP Cache Poisoning

  • Continuous fake ARP messages
  • Poison ARP tables of multiple hosts
  • Intercept network traffic
  • Modify or steal data

Other ARP-Based Attacks

  • ARP Flooding: Overwhelm switches with fake ARP requests
  • DoS via ARP: Map legitimate IP to non-existent MAC
  • MAC Cloning: Impersonate another device's MAC address

Attack Steps

  1. Attacker sends fake ARP reply to Victim: "192.168.1.1 is at [Attacker's MAC]"
  2. Victim updates ARP cache with false gateway mapping
  3. Victim sends all internet-bound traffic to Attacker
  4. Attacker intercepts, reads/modifies, then forwards to real gateway
  5. Neither Victim nor Gateway knows they're being intercepted

Attack Impact

Complete network traffic interception, credential theft, session hijacking, data manipulation

IPv6 Equivalent - NDP

Neighbor Discovery Protocol (NDP)

IPv6 does not use ARP. Instead, it uses NDP (Neighbor Discovery Protocol) which is part of ICMPv6.

ARP vs NDP Comparison

Feature ARP (IPv4) NDP (IPv6)
Protocol Standalone (EtherType 0x0806) ICMPv6 (Type 135, 136)
Request Message ARP Request (broadcast) Neighbor Solicitation (multicast)
Reply Message ARP Reply (unicast) Neighbor Advertisement (unicast)
Broadcast Uses broadcast (FF:FF:FF:FF:FF:FF) Uses multicast (more efficient)
Security No built-in security IPSec support, SEND protocol
Duplicate Detection Gratuitous ARP (informal) DAD (Duplicate Address Detection)

Troubleshooting ARP Issues

Common Problems & Solutions

Problem: Cannot Communicate with Local Device

  • Check: ARP cache for correct mapping (arp -a)
  • Check: Duplicate IP addresses on network
  • Test: Clear ARP cache and retry
  • Test: Ping device to trigger new ARP request
  • Verify: No firewall blocking ARP traffic

Problem: Incomplete ARP Entries

  • Cause: Destination not responding to ARP requests
  • Check: Device is powered on and connected
  • Check: No VLAN mismatches
  • Check: Firewall not blocking traffic

Problem: Suspected ARP Spoofing

  • Detect: Monitor for duplicate MAC addresses with different IPs
  • Detect: Watch for frequent ARP cache changes
  • Prevent: Use static ARP entries for critical devices
  • Prevent: Enable Dynamic ARP Inspection (DAI) on switches
  • Prevent: Use port security features

ARP Quick Reference

Key Facts

  • Purpose: Map IP to MAC
  • EtherType: 0x0806
  • Request: Broadcast
  • Reply: Unicast
  • RFC: 826
  • Scope: Local network only
  • Cache Timeout: 2-20 min (varies)

Must-Know Commands

  • arp -a - View cache
  • arp -d [ip] - Delete entry
  • ip neigh show - Linux view
  • show arp - Cisco view
  • clear arp-cache - Cisco clear

Security Reminders

  • ARP has NO authentication - vulnerable to spoofing
  • Enable Dynamic ARP Inspection (DAI) on switches
  • Use static ARP entries for critical infrastructure
  • Monitor for suspicious ARP traffic
  • Consider 802.1X authentication for network access control

Slide 16

Summary

What We Learned

  • ARP bridges Layer 2 and Layer 3 by mapping IP addresses to MAC addresses
  • ARP uses broadcast requests and unicast replies
  • ARP cache stores mappings to reduce network traffic
  • Gratuitous ARP announces IP/MAC mappings without being asked
  • Proxy ARP allows routers to respond on behalf of other devices
  • ARP only works on local networks; remote traffic uses gateway MAC
  • ARP has serious security vulnerabilities (spoofing, poisoning)
  • IPv6 uses NDP instead of ARP, with improved security

Key Security Takeaway

ARP was designed in 1982 for trusted networks. Modern networks require additional security measures like DAI, port security, and 802.1X to protect against ARP-based attacks.

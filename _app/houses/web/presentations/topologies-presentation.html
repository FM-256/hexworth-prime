Physical vs Logical Topology

Physical Topology

The actual physical layout of cables, devices, and equipment.

What you can SEE and TOUCH:
  • Where cables are routed
  • Physical location of switches
  • Server room layout
  • Patch panel connections

Example: Cables from every desk run to a central wiring closet (physical star)

Logical Topology

How data actually flows through the network, regardless of physical layout.

How data MOVES:
  • Data flow paths
  • Network addressing schemes
  • VLAN configurations
  • Routing decisions

Example: Token Ring (logical ring) can run on physical star wiring

Key Insight

Physical and logical topologies can be DIFFERENT! A network might be physically wired as a star (all cables to central switch) but logically function as a bus (Ethernet's CSMA/CD).

Star Topology

How It Works

All devices connect to a central device (switch, hub, or router). No direct connections between endpoints.

Advantages

  • Easy to install & manage
  • One failure doesn't affect others
  • Easy to add new devices
  • Easy troubleshooting

Disadvantages

  • Central device = single point of failure
  • More cabling required
  • Performance depends on central device
Most Common Topology! Nearly all modern LANs use star topology. Each PC connects to a switch, switches connect to distribution switches, which connect to core switches.

Extended Star / Hub-and-Spoke

Multiple Stars Connected

An extended star connects multiple star topologies together through intermediate devices.

Also Called:

Bus Topology

How It Works

All devices connect to a single backbone cable. Data travels in both directions. Terminators prevent signal reflection.

Advantages

  • Simple, inexpensive
  • Easy to extend
  • Less cabling than star

Disadvantages

  • Single point of failure (backbone)
  • Difficult to troubleshoot
  • Limited cable length
  • Performance degrades with more devices
  • Collisions occur
Legacy Technology! Bus topology using coaxial cable (10BASE2, 10BASE5) is rarely used in modern networks. However, the logical bus concept lives on in Ethernet's CSMA/CD!

Ring Topology

How It Works

Each device connects to exactly two neighbors, forming a closed loop. Data travels in one direction (or both in dual-ring).

Token Passing:
  • A "token" circulates around the ring
  • Only the device with the token can transmit
  • Eliminates collisions!
  • Used by Token Ring and FDDI

Advantages

  • No collisions
  • Predictable performance
  • Equal access for all

Disadvantages

  • One break = entire network down
  • Difficult to troubleshoot
  • Adding devices disrupts network
Historical Note: IBM's Token Ring was popular in the 1980s-90s but lost to Ethernet. FDDI (Fiber Distributed Data Interface) uses dual counter-rotating rings for redundancy.

Mesh Topology

Full Mesh

Every device connected to every other

Partial Mesh

Critical nodes have multiple paths

Full Mesh Formula

Number of connections needed: n(n-1)/2 where n = number of devices

5 devices = 5(4)/2 = 10 connections | 10 devices = 10(9)/2 = 45 connections!

Advantages

  • Maximum redundancy
  • Multiple paths = fault tolerant
  • No single point of failure

Disadvantages

  • Very expensive (cables + ports)
  • Complex to manage
  • Not scalable (connections grow exponentially)

Hybrid Topology

A hybrid topology combines two or more different topologies to meet specific requirements.

Common Hybrid Examples

Star-Bus Hybrid:

Multiple star networks connected by a bus backbone. Common in older office buildings.

Star-Ring Hybrid:

Star-wired physical topology running Token Ring logical protocol.

Star-Mesh Hybrid (Most Common!):

Star topology at the access layer, partial mesh at distribution/core for redundancy.

Enterprise Example

Core=Mesh, Distribution=Partial Mesh, Access=Star

Why Hybrid?

Different parts of a network have different needs. End-user access needs simplicity (star), while core networks need redundancy (mesh). Hybrid gives you the best of both worlds!

Storage & Data Center Networks

SAN - Storage Area Network

Definition: High-speed network dedicated to storage devices
  • Purpose: Connect servers to storage arrays
  • Protocols: Fibre Channel, iSCSI, FCoE
  • Speed: 8-32 Gbps (Fibre Channel)
  • Benefit: Separates storage traffic from LAN
Block-Level Access: SAN provides raw block storage - the server sees it as a local disk, not a network share.

NAS vs SAN

Feature NAS SAN
Access File-level Block-level
Protocol NFS, SMB/CIFS FC, iSCSI
Network Ethernet (LAN) Dedicated SAN
Cost Lower Higher
Use Case File shares Databases, VMs

NAS vs SAN Analogy

NAS: Like accessing files on a shared folder - you browse and open files by name.
SAN: Like having an external hard drive plugged directly into your computer - it appears as drive letter D:, and the server manages the filesystem.

SD-WAN & Software-Defined Networking

SD-WAN

Software-Defined Wide Area Network

What It Does:
  • Abstracts WAN hardware from control
  • Centralized management of multiple WAN links
  • Can combine MPLS, Internet, 4G/5G
  • Automatic path selection based on conditions
Benefits:
  • Reduced WAN costs (use cheaper Internet)
  • Improved performance (dynamic routing)
  • Simplified management (centralized)
  • Better security (encrypted tunnels)

SDN - Software-Defined Networking

Key Concept: Separate the control plane from the data plane
  • Control Plane: Centralized SDN controller makes decisions
  • Data Plane: Switches just forward packets as instructed
  • Protocol: OpenFlow communicates between controller and switches
Traditional vs SDN:

Traditional: Each switch/router has its own brain (distributed control)

SDN: One centralized brain controls all switches (centralized control)

Why This Matters for Exams

SD-WAN and SDN are increasingly tested on Network+ and CCNA. Know that they represent the shift from hardware-centric to software-centric networking, enabling automation, programmability, and centralized management.

Specialized Network Types

MPLS Network

Multiprotocol Label Switching
  • High-performance WAN technology
  • Uses labels instead of IP lookups
  • Provider-managed, guaranteed SLAs
  • Supports QoS for voice/video
  • Being replaced by SD-WAN in some cases

mGRE / DMVPN

Dynamic Multipoint VPN
  • Creates mesh VPN over Internet
  • Spoke-to-spoke direct tunnels
  • Reduces hub bandwidth requirements
  • Cost-effective WAN alternative

ICS/SCADA Networks

Industrial Control Systems
  • Control physical processes (factories, utilities)
  • SCADA: Supervisory Control and Data Acquisition
  • Often isolated "air-gapped" networks
  • Legacy protocols (Modbus, DNP3)
  • Security critical! (e.g., power grid)

IoT Networks

Internet of Things
  • Connects sensors, devices, actuators
  • Protocols: MQTT, CoAP, Zigbee, Z-Wave
  • Often low-power, low-bandwidth
  • Security challenges (many devices)

Real-World Examples

ICS/SCADA: Water treatment plant, power grid, manufacturing robots
IoT: Smart thermostat, fitness tracker, connected car

Three-Tier Network Architecture

Core Layer
  • High-speed backbone
  • Connects distribution switches
  • No access control here
  • Redundant, fast (40-100 Gbps)
Distribution Layer
  • Policy enforcement (ACLs, QoS)
  • Inter-VLAN routing
  • Aggregates access switches
  • Redundant uplinks to core
Access Layer: Connects end devices (PCs, phones, printers). Port security, VLANs, PoE for phones/APs.
Slide 14 of 18 Network Essentials

Spine-Leaf (Data Center) Architecture

Modern Data Center Design

Key Feature: Every LEAF connects to every SPINE (full mesh between tiers)
  • Predictable latency: Max 2 hops between any servers
  • Easy scaling: Add more leaves or spines
  • No STP needed: Uses ECMP (Equal-Cost Multi-Path)
  • East-West traffic: Optimized for server-to-server
Used by: Cloud providers, large data centers, virtualized environments

Three-Tier vs Spine-Leaf

Three-Tier: Best for traditional enterprise (more North-South traffic to Internet)
Spine-Leaf: Best for data centers (more East-West traffic between servers)

Quick Check: Your company has 100 remote offices connecting to HQ. Which topology?

Answer: Hub-and-Spoke (Extended Star) - HQ is the hub, offices are spokes. Cost-effective, centralized management. Add partial mesh between regional hubs for redundancy.

Key Exam Points

Network+ N10-008 Focus

Q: What's the main disadvantage of star topology?

A: Single point of failure (central device)

Q: How many connections for full mesh with 6 nodes?

A: n(n-1)/2 = 6(5)/2 = 15 connections

Q: Physical star, logical bus - what technology?

A: Ethernet (CSMA/CD is logically a bus)

CCNA 200-301 Focus

Q: Three-tier architecture layers?

A: Core, Distribution, Access

Q: Spine-leaf is best for what traffic pattern?

A: East-West (server-to-server in data centers)

Q: SD-WAN's main benefit?

A: Centralized management, use multiple WAN links (including cheaper Internet)

Memory Tips

  • Star: Think "star" shape - everything points to center
  • Bus: Like a city bus - one route, everyone gets on/off
  • Ring: Olympic rings - connected in a circle
  • Mesh: Spider web - everything connected

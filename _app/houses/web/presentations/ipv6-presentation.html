IPv6 Prefix Notation

Like IPv4 CIDR, IPv6 uses slash notation to indicate the network prefix length:

/64

Common Prefix Lengths

Prefix Usage Bits for Hosts
/64 Standard subnet (LAN) 64 bits = 18 quintillion hosts
/48 Site allocation 65,536 /64 subnets
/128 Single host (loopback) 0 bits = 1 host only
Remember: The standard subnet size for LANs is /64. This allows SLAAC to work properly with EUI-64.

IPv6 Address Compression Rules

IPv6 addresses can be shortened using two rules:

Rule 1: Omit Leading Zeros

Remove leading zeros in each hextet (but keep at least one digit)

2001:0db8:0000:0001 ‚Üí 2001:db8:0:1

Rule 2: Double Colon (::)

Replace ONE contiguous group of all-zero hextets with :: (only once per address!)

2001:db8:0:0:0:0:0:1 ‚Üí 2001:db8::1
‚ö†Ô∏è Important: You can only use :: once per address! Using it twice creates ambiguity.

IPv6 Address Types Overview

IPv6 defines several address types based on their scope and purpose:

Type Prefix Scope
Global Unicast (GUA) 2000::/3 Internet routable
Link-Local (LLA) fe80::/10 Single link only
Unique Local (ULA) fc00::/7 Private (like RFC 1918)
Multicast ff00::/8 One-to-many
Anycast (same as unicast) One-to-nearest
Note: IPv6 has NO broadcast addresses! Multicast replaces broadcast functionality.

Link-Local Addresses (LLA)

Prefix: fe80::/10 (you'll typically see fe80::)

Scope: Valid only on the local link - NOT routable!

Key Characteristics

  • Automatically assigned when IPv6 is enabled on an interface
  • Required on every IPv6-enabled interface
  • Used for: Neighbor discovery, routing protocol traffic, next-hop addresses
  • Never routed beyond the local network segment
Example: fe80::1 Example: fe80::211:22ff:fe33:4455
‚ö†Ô∏è Remember: When using link-local addresses for communication (like ping), you must specify the outgoing interface because the same LLA could exist on multiple interfaces!

Unique Local Addresses (ULA)

Prefix: fc00::/7 (commonly fd00::/8 is used)

Equivalent to: IPv4 private addresses (10.x, 172.16-31.x, 192.168.x)

When to Use ULA

  • Internal network communication that doesn't need internet access
  • Devices that should never be directly reachable from the internet
  • Organizations that want consistent internal addressing

Multicast Addresses

Prefix: ff00::/8

Purpose: Send packets to multiple destinations simultaneously

Common Multicast Addresses

Address Scope Description
ff02::1 Link-local All IPv6 nodes
ff02::2 Link-local All IPv6 routers
ff02::5 Link-local All OSPF routers
ff02::6 Link-local All OSPF DRs
ff02::1:ff00:0/104 Link-local Solicited-node multicast
Key Concept: Multicast replaces broadcast in IPv6. Instead of ff:ff:ff:ff:ff:ff, use ff02::1 to reach all nodes on a link.

Format: Same as unicast addresses

IPv6 Static Configuration

Manually configuring IPv6 addresses on Cisco devices:

Router Interface Configuration

Router(config)# interface GigabitEthernet0/0 Router(config-if)# ipv6 address 2001:db8:acad:1::1/64 Router(config-if)# ipv6 address fe80::1 link-local Router(config-if)# no shutdown

Enable IPv6 Routing

Router(config)# ipv6 unicast-routing

EUI-64: Interface ID Generation

EUI-64 creates a 64-bit Interface ID from a device's 48-bit MAC address:

The EUI-64 Process

MAC Address
00:11:22:33:44:55
‚Üí
Split in half
00:11:22 | 33:44:55
‚Üí
Insert FF:FE
00:11:22:FF:FE:33:44:55
‚Üí
Flip 7th bit
02:11:22:FF:FE:33:44:55
MAC: 00:11:22:33:44:55 ‚Üì EUI-64: 0211:22ff:fe33:4455 ‚Üì Full IPv6: 2001:db8:1::0211:22ff:fe33:4455
Privacy Concern: EUI-64 embeds the MAC address, which can be used to track devices. Many modern OSes use random Interface IDs instead (Privacy Extensions - RFC 4941).

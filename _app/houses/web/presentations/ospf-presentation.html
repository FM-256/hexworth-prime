What is OSPF?

Definition

OSPF is a link-state routing protocol that uses the Dijkstra algorithm to calculate the shortest path to each destination network.

Key Characteristics

  • Protocol: IP Protocol 89 (not TCP/UDP)
  • Algorithm: Dijkstra's Shortest Path First (SPF)
  • Metric: Cost (based on bandwidth)
  • Administrative Distance: 110
  • Multicast Addresses: 224.0.0.5 (All OSPF Routers), 224.0.0.6 (DR/BDR)
  • Classless: Supports VLSM and CIDR
  • Authentication: Plain text, MD5, SHA (OSPFv3)

6. Loading

Neighbors fully adjacent, databases synchronized

Slide 5

DR/BDR - Designated Router Election

What is DR/BDR?

DR (Designated Router) and BDR (Backup Designated Router) are elected routers on broadcast and non-broadcast multi-access (NBMA) networks that reduce OSPF overhead.

Why DR/BDR?

  • Problem: On multi-access networks (Ethernet), every router forming adjacencies with every other router creates n(n-1)/2 adjacencies
  • Solution: All routers form adjacencies only with the DR and BDR
  • Benefit: Reduces LSA flooding traffic and database synchronization overhead

Election Process

  • Priority: Router with highest OSPF priority wins (default = 1, range 0-255)
  • Tiebreaker: Highest Router ID becomes DR, second-highest becomes BDR
  • Priority 0: Router cannot become DR/BDR (useful for low-power devices)
  • Non-Preemptive: DR/BDR election only occurs when network comes up or DR/BDR fails

Multicast Addresses

  • 224.0.0.5 (AllSPFRouters): All OSPF routers listen
  • 224.0.0.6 (AllDRouters): Only DR and BDR listen

LSA Flooding

LSA Types

  • Type 1 - Router LSA: Lists all router's links and costs
  • Type 2 - Network LSA: Generated by DR for multi-access networks
  • Type 3 - Summary LSA: ABR sends between areas
  • Type 4 - ASBR Summary: Describes route to ASBR
  • Type 5 - External LSA: External routes redistributed into OSPF
  • Type 7 - NSSA External: External routes within NSSA

Slide 8

Designated Router (DR) / Backup DR

Why DR/BDR?

On multi-access networks (Ethernet), DR/BDR reduce the number of adjacencies and LSA flooding overhead.

Election Process

  • Highest priority wins (default: 1, range: 0-255)
  • Priority 0 = never become DR/BDR
  • If priority tied, highest Router ID wins
  • Router ID: Highest loopback IP > Highest active interface IP
  • Non-preemptive: Existing DR/BDR stay until failure

‚ö† Important Note

DR/BDR election is NOT preemptive. If you want a specific router to be DR, configure it first or clear OSPF process on other routers.

OSPF Cost Metric

Cost Formula

Cost = Reference Bandwidth / Interface Bandwidth

Default Reference Bandwidth: 100 Mbps (100,000,000 bps)

Interface Type Bandwidth Default Cost
Serial (T1) 1.544 Mbps 64
Ethernet 10 Mbps 10
Fast Ethernet 100 Mbps 1
Gigabit Ethernet 1 Gbps 1
10 Gigabit Ethernet 10 Gbps 1

‚ö† Problem with Default Reference Bandwidth

FastEthernet, GigE, and 10GigE all have cost of 1! Adjust reference bandwidth for modern networks.

OSPF Troubleshooting

Common Issues & Solutions

‚ö† Neighbors Not Forming

  • Check: Hello/Dead intervals match
  • Check: Area ID matches
  • Check: Subnet masks match on broadcast networks
  • Check: Authentication matches
  • Check: No ACLs blocking protocol 89 or multicast
  • Check: Interface not passive on both ends

‚ö† Stuck in ExStart/Exchange State

  • Cause: MTU mismatch between neighbors
  • Solution: Ensure MTU matches on both interfaces
  • Workaround: Use ip ospf mtu-ignore
Prepared by: EQ6
Slide 14

OSPF Troubleshooting (Continued)

‚ö† Routes Not Appearing in Routing Table

  • Check: Neighbors in Full state (show ip ospf neighbor)
  • Check: LSAs present in database (show ip ospf database)
  • Check: Area 0 connectivity (all areas must connect to backbone)
  • Check: No discard routes or summarization issues
  • Check: Administrative distance (OSPF = 110)

‚ö† High CPU Usage

  • Cause: Too many routes, frequent topology changes, SPF recalculation
  • Solution: Implement areas, use route summarization
  • Solution: Tune SPF timers (timers throttle spf)
  • Solution: Use passive interfaces where appropriate

Slide 15

OSPF Best Practices

‚úÖ DO

  • Use loopback interfaces for Router IDs
  • Implement hierarchical design with areas
  • Use route summarization at area boundaries
  • Configure authentication for security
  • Adjust reference bandwidth for modern networks (10000 or 100000)
  • Use passive interfaces on user-facing ports
  • Document Router IDs and area design

‚ùå DON'T

  • Run OSPF on WAN links without tuning timers
  • Create areas without connecting to Area 0
  • Use default reference bandwidth with GigE+ links
  • Ignore MTU mismatches
  • Skip authentication in production

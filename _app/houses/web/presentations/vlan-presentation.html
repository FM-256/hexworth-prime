Problems VLANs Solve

  • Broadcast Domain Size: Limit broadcast traffic to reduce network congestion
  • Security: Isolate sensitive data from general network traffic
  • Network Management: Logical grouping without physical constraints
  • Cost Efficiency: One switch supports multiple logical networks
  • Flexibility: Move users between departments without rewiring

The Core Problem

Before VLANs, every department needed separate physical switches and infrastructure. VLANs enable logical separation on the same physical hardware.

What is a VLAN?

Definition

A VLAN (Virtual Local Area Network) is a logical broadcast domain that can span multiple physical LAN segments. VLANs partition a network at Layer 2.

IEEE 802.1Q Standard

  • Industry Standard: Vendor-neutral VLAN tagging protocol
  • Tag Size: 4 bytes inserted into Ethernet frame
  • VLAN Range: 1-4094 (0 and 4095 reserved)
  • Frame Modification: Inserts tag between Source MAC and EtherType
  • Backward Compatible: Untagged frames still supported (native VLAN)

Key Characteristics

  • Layer 2 Technology: Operates at Data Link Layer
  • Broadcast Isolation: Each VLAN = separate broadcast domain
  • Inter-VLAN Communication: Requires Layer 3 device (router/L3 switch)
  • Default VLAN: VLAN 1 (cannot be deleted on Cisco switches)

Slide 4

Access Ports vs Trunk Ports

Access Ports

  • Purpose: Connect end devices (PCs, printers, phones)
  • VLAN Assignment: Single VLAN only
  • Frame Tagging: Removes tags before sending, adds when receiving
  • Configuration: Static VLAN assignment
  • Use Case: User-facing ports
Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10

Trunk Ports

  • Purpose: Connect switches to switches or routers
  • VLAN Assignment: Carries multiple VLANs
  • Frame Tagging: Adds 802.1Q tags to identify VLANs
  • Configuration: Allows specified VLANs
  • Use Case: Inter-switch links
Switch(config-if)# switchport mode trunk Switch(config-if)# switchport trunk allowed vlan 10,20,30

Important Distinction

Access Port: End device sees normal Ethernet frames (no tags)
Trunk Port: Interconnected devices see tagged frames (802.1Q headers)

802.1Q Tag Fields

Field Size Description
TPID (Tag Protocol ID) 16 bits Value: 0x8100 - Identifies frame as 802.1Q tagged
PCP (Priority Code Point) 3 bits 802.1p CoS priority (0-7)
DEI (Drop Eligible) 1 bit Can frame be dropped during congestion?

Frame Size Impact

802.1Q adds 4 bytes to the frame. Maximum frame size increases from 1518 to 1522 bytes (baby giant frame).

Native VLAN Concept

What is Native VLAN?

The native VLAN is the VLAN that carries untagged traffic on a trunk port. By default, it is VLAN 1.

No tag (untagged)

Native VLAN Behavior

  • Sending: Frames from native VLAN sent without 802.1Q tag
  • Receiving: Untagged frames placed in native VLAN
  • Default: VLAN 1 (change for security)
  • Must Match: Native VLAN must match on both ends of trunk

Security Best Practice

Change native VLAN from default VLAN 1 to unused VLAN to prevent VLAN hopping attacks and management traffic exposure.

VLAN Trunking Protocol (VTP)

What is VTP?

VTP is a Cisco proprietary protocol that propagates VLAN configuration across switches automatically.

VTP Modes

Mode Can Create VLANs? Forwards Advertisements? Synchronizes?
Server Yes Yes Yes
Client No Yes Yes
Transparent Yes (local only) Yes No
Off (VTPv3) Yes (local only) No No

VTP Danger Zone

  • Higher Revision Number Wins: Can accidentally wipe all VLANs
  • Domain Name: Must match for synchronization
  • Best Practice: Use VTP Transparent or Off mode in most networks
  • Password: Use VTP password for added security

Inter-VLAN Routing Methods

Why Inter-VLAN Routing?

VLANs are Layer 2 constructs and cannot communicate with each other directly. A Layer 3 device is required to route between VLANs.

Legacy Method

One Cable Per VLAN

Separate physical interface for each VLAN

Wasteful, doesn't scale

Router-on-a-Stick

Subinterfaces

Single physical link with logical subinterfaces

Good for small networks

Layer 3 Switch

SVI (Switch Virtual Interface)

Switch performs routing in hardware

Best performance, modern standard

Method Comparison

Method Performance Scalability Cost Use Case
Legacy (separate links) Low Very Poor High Obsolete
Router-on-a-Stick Moderate Moderate Low Small networks, branch offices
Layer 3 Switch (SVI) High Excellent Moderate Enterprise networks, data centers

SVI (Switch Virtual Interface) Configuration

What is an SVI?

An SVI is a virtual Layer 3 interface on a multilayer switch that represents a VLAN. It enables the switch to route between VLANs at wire speed.

  • Explicitly configure trunk mode (don't rely on auto/desirable)
  • Change native VLAN from default VLAN 1
  • Use switchport trunk allowed vlan to limit VLANs
  • Disable DTP with switchport nonegotiate

Troubleshooting Common VLAN Issues

Issue: Devices in Same VLAN Cannot Communicate

  • Check: Port assigned to correct VLAN (show vlan brief)
  • Check: Port in correct mode (show interfaces switchport)
  • Check: VLAN exists on all switches in path
  • Check: Trunk links carrying the VLAN (show interfaces trunk)
  • Check: IP addresses in same subnet

Issue: Devices in Different VLANs Cannot Communicate

  • Check: Inter-VLAN routing configured (router or L3 switch)
  • Check: Default gateway configured on end devices
  • Check: Router/L3 switch has IP addresses in both VLANs
  • Check: ip routing enabled on L3 switch
  • Check: No ACLs blocking traffic between VLANs

Issue: Native VLAN Mismatch

  • Symptom: CDP/LLDP warnings, connectivity issues
  • Solution: Ensure native VLAN matches on both ends of trunk
  • Verify: show interfaces trunk

VLAN & Trunking Best Practices

Design Best Practices

  • VLAN 1: Don't use for user data; reserve for management or disable
  • Native VLAN: Change from default; use unused VLAN (e.g., 999)
  • VLAN Naming: Use descriptive names matching function/department
  • VLAN Ranges: Plan and document VLAN numbering scheme
  • Separate VLANs: Voice, data, management, guest on different VLANs
  • Limit VLAN Size: Keep broadcast domains reasonably sized

Security Best Practices

  • Unused Ports: Place in shutdown state and unused VLAN
  • Trunk Pruning: Only allow necessary VLANs on trunks
  • Disable DTP: Use switchport nonegotiate
  • VTP Mode: Use Transparent or Off to prevent VLAN database corruption
  • Port Security: Implement on access ports to prevent MAC flooding
  • Private VLANs: Use for additional isolation in same IP subnet

Documentation

Maintain VLAN database documentation: VLAN IDs, names, IP subnets, and purpose

VLAN Quick Reference

Key Facts

  • Standard: IEEE 802.1Q
  • VLAN Range: 1-4094
  • Normal Range: 1-1005
  • Extended Range: 1006-4094
  • Tag Size: 4 bytes
  • Default VLAN: 1
  • Reserved: 0, 4095

Essential Commands

  • switchport mode access
  • switchport access vlan [id]
  • switchport mode trunk

Slide 17

Summary

What We Learned

  • VLANs create logical broadcast domains at Layer 2
  • 802.1Q is the industry-standard VLAN tagging protocol
  • Access ports connect end devices to single VLAN (untagged)
  • Trunk ports carry multiple VLANs between switches (tagged)
  • Native VLAN carries untagged traffic on trunk (default: VLAN 1)
  • Inter-VLAN routing requires Layer 3 device (router or L3 switch)
  • Router-on-a-stick uses subinterfaces for small networks
  • SVIs provide wire-speed routing on Layer 3 switches
  • VTP propagates VLAN configuration but use with caution
  • Security requires proper VLAN design and trunk configuration

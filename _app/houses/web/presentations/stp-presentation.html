STP History & The Problem It Solves

1985

Radia Perlman invents STP algorithm

1990

IEEE 802.1D Standard published

The Problem: Layer 2 Loops

  • Without STP: Redundant switch links create loops
  • Broadcast storms: Frames loop infinitely, consuming all bandwidth
  • MAC table instability: Switches can't learn stable MAC addresses
  • Multiple frame copies: Hosts receive duplicate frames

What is STP?

Definition

STP is a Layer 2 protocol that prevents loops in switched networks by logically blocking redundant paths while maintaining them as backup links.

Key Characteristics

  • Standard: IEEE 802.1D (original), 802.1w (RSTP), 802.1s (MSTP)
  • Function: Creates loop-free logical topology
  • Mechanism: BPDUs (Bridge Protocol Data Units)
  • Default Timer: Hello 2 seconds, Max Age 20 seconds, Forward Delay 15 seconds
  • Convergence Time: 30-50 seconds (802.1D), 1-6 seconds (RSTP)
  • Destination MAC: 01:80:C2:00:00:00 (multicast)

‚ö† Important

All Cisco switches run STP by default. It cannot be disabled globally, only per-VLAN.

Slide 3

STP Election Process (3 Steps)

Step 1: Elect Root Bridge

Root Bridge is the reference point for the spanning tree topology.

  • Switch with lowest Bridge ID becomes Root Bridge
  • Bridge ID = Priority (default 32768) + MAC Address
  • Lower priority wins; if tied, lower MAC wins

Step 2: Elect Root Ports

On each non-root switch, select the port with the best path to Root Bridge.

  • Based on path cost (cumulative to Root)
  • One Root Port per non-root switch

Step 3: Elect Designated Ports

On each network segment, elect the port that forwards traffic toward Root Bridge.

  • Switch with lowest cost to Root has Designated Port on that segment
  • All Root Bridge ports are Designated Ports

üí° Result

All other ports become Blocked (Alternate) ports - preventing loops!

Bridge ID and Priority

Bridge ID Components

Component Size Description
Priority 4 bits 0-61440 in increments of 4096
Extended System ID 12 bits VLAN ID (PVST+)
MAC Address 48 bits Switch base MAC address

Priority Values

  • Default: 32768 (+ VLAN ID with PVST+)
  • Valid range: 0, 4096, 8192, 12288, 16384, 20480, 24576, 28672, 32768, 36864, 40960, 45056, 49152, 53248, 57344, 61440
  • Lower is better: Priority 0 = always becomes Root (if lowest MAC)

STP Port States (802.1D)

Disabled

Administratively down

Time: N/A

Blocking

Receives BPDUs only

Time: 20 sec

Listening

Sends/receives BPDUs

Time: 15 sec

Learning

Builds MAC table

Time: 15 sec

Forwarding

Forwards traffic

Active

State Transitions

  • Blocking ‚Üí Listening: Port selected as Root or Designated
  • Listening ‚Üí Learning: After 15 seconds (Forward Delay)
  • Learning ‚Üí Forwarding: After 15 seconds (Forward Delay)
  • Total Time: 30-50 seconds to reach Forwarding state

‚ö† Convergence Time

802.1D STP takes 30-50 seconds to converge. This is why RSTP (802.1w) was created!

Slide 7

Bridge Protocol Data Units (BPDUs)

BPDU

BPDU Contents

  • Root Bridge ID: Who is the Root?
  • Cost to Root: Sender's path cost to Root
  • Sender Bridge ID: Who sent this BPDU?
  • Sender Port ID: Which port sent this?
  • Timers: Hello, Max Age, Forward Delay
  • Flags: Topology Change, TC Acknowledgment

BPDU Behavior

  • Root Bridge: Sends BPDUs every 2 seconds (Hello Time)
  • Non-Root Switches: Relay BPDUs (update cost value)
  • Blocked Ports: Receive BPDUs, don't forward
  • Max Age (20s): If no BPDU received, assume link/switch failed

Slide 8

RSTP - Rapid Spanning Tree (802.1w)

Why RSTP?

802.1D STP is too slow for modern networks (30-50 second convergence)

RSTP Improvements

  • Faster Convergence: 1-6 seconds (vs 30-50 seconds)
  • New Port Roles: Root, Designated, Alternate, Backup
  • New Port States: Discarding, Learning, Forwarding (3 vs 5 states)
  • Edge Ports: Immediate forwarding (like PortFast)
  • Point-to-Point Links: Fast transition between switches
  • Proposal/Agreement: Rapid synchronization mechanism

802.1D (STP)

  • 5 port states
  • Passive convergence
  • 30-50 second failover
  • Timer-based

802.1w (RSTP)

  • 3 port states
  • Active proposal/agreement
  • 1-6 second failover
  • Handshake-based

Slide 9

PortFast & BPDU Guard

PortFast

Purpose

Immediately transitions port to Forwarding state (bypasses Listening and Learning)

  • Use Case: End devices (PCs, servers, printers)
  • Benefit: Device gets network access immediately (no 30-second wait)
  • Warning: NEVER use on ports connected to switches!
! Enable PortFast on an access port Switch(config)# interface GigabitEthernet0/1 Switch(config-if)# spanning-tree portfast ! Enable globally on all access ports Switch(config)# spanning-tree portfast default

BPDU Guard

‚ö† Protection Mechanism

Shuts down port if BPDU is received (prevents accidental loops)

! Enable BPDU Guard on PortFast ports Switch(config)# interface GigabitEthernet0/1 Switch(config-if)# spanning-tree bpduguard enable ! Enable globally on all PortFast ports Switch(config)# spanning-tree portfast bpduguard default

Slide 10

Root Guard & Loop Guard

Root Guard

Purpose

Prevents another switch from becoming Root Bridge on a specific port

  • Use Case: Ports facing access layer or customer equipment
  • Behavior: If superior BPDU received, port goes into "root-inconsistent" state
  • Recovery: Automatic when superior BPDUs stop
Switch(config)# interface GigabitEthernet0/1 Switch(config-if)# spanning-tree guard root

Loop Guard

Purpose

Prevents alternate/blocked ports from transitioning to Forwarding due to unidirectional link failure

  • Use Case: Point-to-point links between switches
  • Behavior: If BPDUs stop on non-designated port, port goes to "loop-inconsistent" state
! Enable globally Switch(config)# spanning-tree loopguard default ! Or per-interface Switch(config-if)# spanning-tree guard loop

STP Troubleshooting

Common Issues & Solutions

‚ö† Issue: Wrong Root Bridge

  • Symptom: Suboptimal Root, traffic taking inefficient paths
  • Cause: Switch with lowest MAC (not best switch) became Root
  • Solution: Manually configure priority on desired Root
  • Command: spanning-tree vlan 1 root primary

‚ö† Issue: Port Stuck in Blocking

  • Symptom: Port should be Forwarding but stuck Blocking
  • Cause: Receiving superior BPDUs, higher cost, or inconsistent state
  • Check: show spanning-tree interface
  • Solution: Verify topology, check for Root Guard violations

‚ö† Issue: Slow Convergence

  • Symptom: 30-50 seconds for network to recover
  • Cause: Using 802.1D STP instead of RSTP
  • Solution: spanning-tree mode rapid-pvst

Slide 14

STP Troubleshooting (Continued)

‚ö† Issue: Port Err-Disabled

  • Symptom: Port shut down, LED amber
  • Cause: BPDU Guard triggered (BPDU received on PortFast port)
  • Check: show interface status err-disabled
  • Recovery: Remove switch, shutdown then no shutdown
  • Prevention: Don't connect switches to PortFast ports!

‚ö† Issue: Broadcast Storm

  • Symptom: Network extremely slow, CPU 100%, switch fans loud
  • Cause: STP disabled or failed, creating Layer 2 loop
  • Emergency: Physically unplug redundant links
  • Solution: Re-enable STP, verify proper Root election
! Recover from err-disabled Switch(config)# interface GigabitEthernet0/1 Switch(config-if)# shutdown Switch(config-if)# no shutdown ! Or enable auto-recovery (after 5 minutes) Switch(config)# errdisable recovery cause bpduguard Switch(config)# errdisable recovery interval 300

Slide 15

STP Best Practices

‚úÖ DO

  • Manually configure Root Bridge - Don't let lowest MAC decide!
  • Use RSTP (rapid-pvst) - Faster convergence
  • Enable PortFast on access ports - Faster end-device connectivity
  • Enable BPDU Guard on PortFast ports - Prevent loops
  • Enable Root Guard on distribution ports - Prevent rogue Root
  • Document your STP design - Know which switch is Root, why
  • Monitor for topology changes - Frequent changes indicate problems

‚ùå DON'T

  • Don't disable STP (even if you think you don't need it!)
  • Don't use PortFast on trunk or switch-facing ports
  • Don't ignore err-disabled ports (investigate root cause)
  • Don't mix STP modes (PVST+, Rapid-PVST, MST) without planning
  • Don't forget about STP when adding new switches

Summary

What We Learned

  • STP is a Layer 2 protocol that prevents loops in switched networks
  • Invented by Radia Perlman in 1985, standardized as IEEE 802.1D
  • Root Bridge election: Lowest Bridge ID (Priority + MAC)
  • Port roles: Root, Designated, Blocked/Alternate
  • 5 port states (STP): Disabled, Blocking, Listening, Learning, Forwarding
  • 3 port states (RSTP): Discarding, Learning, Forwarding
  • BPDUs: Sent every 2 seconds to maintain topology
  • PortFast: Immediately forward on access ports
  • BPDU Guard: Protect against loops from user-connected switches
  • RSTP (802.1w): Fast convergence (1-6 seconds vs 30-50 seconds)

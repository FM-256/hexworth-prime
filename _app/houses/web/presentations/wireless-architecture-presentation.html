Autonomous Access Points

The original wireless architecture - each AP is a standalone device.

‚úÖ Characteristics

  • Full IOS: Each AP runs complete Cisco IOS software
  • Standalone: Operates independently without controller
  • SSID Configuration: Each AP configured individually
  • Security: WPA2/WPA3 configured per AP
  • Management: CLI, GUI, or SNMP per device

‚ùå Challenges at Scale

  • Configure 100 APs = 100 separate configurations
  • SSID change requires touching every AP
  • No centralized roaming coordination
  • Difficult to maintain consistent security policies
  • No central view of network health
When to Use: Small deployments (1-10 APs), home offices, small retail locations, or when budget is extremely limited.

Split-MAC Architecture

Functions are divided between the AP and WLC:

Performed at AP (Real-Time) Performed at WLC (Management)
802.11 beacons and probes Authentication (802.1X/RADIUS)
RF transmission/reception Association/reassociation
Packet queuing and prioritization Frame translation to 802.3
MAC filtering (if enabled) Security policy enforcement
Encryption/decryption QoS policies
- RF management (power, channel)
- Client roaming coordination
Why Split This Way? Time-sensitive functions (beacons, RF) must happen at the AP. Management functions benefit from centralization for consistency and easier administration.

CAPWAP: Control and Provisioning of Wireless Access Points

The protocol that connects lightweight APs to the WLC.

üîê Control Tunnel

UDP 5246

  • Always encrypted (DTLS)
  • AP configuration
  • Firmware updates
  • Management messages
  • AP keepalives

üìä Data Tunnel

UDP 5247

  • Optional encryption
  • Client data packets
  • Tunneled to WLC
  • WLC forwards to network
  • Can be local switched
Legacy Note: Older Cisco systems used LWAPP (Lightweight AP Protocol) on UDP 12222/12223. CAPWAP is the modern standard (RFC 5415/5416).
! CAPWAP ports on firewall/ACL permit udp any any eq 5246 ! Control tunnel (DTLS encrypted) permit udp any any eq 5247 ! Data tunnel (optional encryption)

How an AP Discovers the WLC

When a lightweight AP powers on, it must find a WLC to join:

DHCP Option 43
DHCP server provides WLC IP address(es) in vendor-specific option 43
DNS Resolution
AP tries to resolve "CISCO-CAPWAP-CONTROLLER.localdomain"
Broadcast Discovery
AP sends broadcast CAPWAP discovery on local subnet
Previously Joined WLC
AP remembers last WLC it joined and tries that first
Static Configuration
Manually configure WLC IP address on AP (rare)
Best Practice: Use DHCP Option 43 in enterprise networks. It's centralized, scalable, and easy to update if WLC IP changes.

FlexConnect (Formerly H-REAP)

A special AP mode that allows local switching when WLC is unreachable.

üü¢ WLC Connected

  • Control tunnel active
  • Data can tunnel to WLC
  • OR switch locally (configured)
  • Centralized auth works
  • Full WLC management

üü° WLC Unreachable

  • AP continues operating
  • Switches traffic locally
  • Uses local authentication
  • Pre-configured SSIDs work
  • No config changes allowed

Perfect For:

  • Branch Offices: WLC at HQ, APs at remote sites over WAN
  • WAN Failures: Branch network keeps working locally
  • Hybrid Deployment: Centralized management, local data
Limitation: When WLC is down, you cannot make configuration changes. The AP operates in a "survive but don't change" mode.

Access Point Operating Modes

Lightweight APs can operate in different modes:

Local Mode

Default mode - provides wireless service to clients while scanning other channels for rogue APs and noise.

Monitor Mode

No client service - dedicated to scanning all channels for rogues, interference, and gathering RF analytics.

FlexConnect

Branch office mode - can switch traffic locally when WLC connection is lost (covered earlier).

Sniffer Mode

Captures all 802.11 packets and forwards to packet analyzer (Wireshark, etc.) for troubleshooting.

Rogue Detector

Listens on wired network to correlate rogue APs detected wirelessly. No RF transmission.

Bridge/Mesh

Point-to-point or point-to-multipoint outdoor wireless links between buildings.

Exam Tip: "Local mode" is default and most common. "FlexConnect" is critical for branch offices. "Monitor" helps with site surveys and rogue detection.

Wireless Roaming

The process of a client moving between APs while maintaining connectivity.

What is Roaming?

When a wireless client moves from one AP to another without dropping the connection. The client disassociates from the old AP and reassociates with the new AP.

üì° Layer 2 Roaming

  • Client moves between APs on same subnet/VLAN
  • IP address stays the same
  • Fast and seamless
  • No re-authentication needed
  • Works with any architecture

üåê Layer 3 Roaming

  • Client moves between APs on different subnets
  • IP address changes (DHCP renew)
  • More complex
  • Requires WLC coordination
  • Only with controller/cloud
Autonomous AP Limitation: Autonomous APs cannot coordinate Layer 3 roaming. Each AP is independent, so moving to a different subnet breaks the session.

Layer 3 Roaming with WLC

How the WLC enables seamless roaming across subnets:

Client Associates to AP1 (VLAN 10)
Client gets IP 10.10.10.50. WLC tracks this as the "anchor" or "home" WLC.
Client Roams to AP2 (VLAN 20)
AP2 is on a different subnet (20.20.20.0/24).
WLC Creates EoIP Tunnel
WLC tunnels traffic from AP2 back to original VLAN 10 (Ethernet-over-IP).
Client Keeps Same IP
Client still uses 10.10.10.50, even though physically on VLAN 20.
Session Maintained
VoIP call, SSH session, etc. continues without interruption.
Key Concept: The WLC "tricks" the client into thinking it never changed subnets. Traffic is tunneled back to the original VLAN so the IP address remains valid.

RF Fundamentals for Wireless

Understanding the radio frequencies used in Wi-Fi:

2.4 GHz Band

2.400 - 2.4835 GHz
  • Channels: 1-11 (US), 1-13 (Europe)
  • Non-overlapping: 1, 6, 11
  • Range: Longer (better penetration)
  • Speed: Lower (max ~300 Mbps)
  • Interference: High (crowded band)

5 GHz Band

5.150 - 5.825 GHz
  • Channels: Many (UNII-1, 2, 3)
  • Non-overlapping: 24+ channels
  • Range: Shorter (less penetration)
  • Speed: Higher (up to 1+ Gbps)
  • Interference: Lower (cleaner)
Best Practice: Use 2.4 GHz for coverage (IoT devices, far corners). Use 5 GHz for capacity (high-speed devices, dense areas). Modern APs support both simultaneously (dual-band).

2.4 GHz Channel Planning

Only 3 non-overlapping channels in the 2.4 GHz band (US):

Channel 1 ‚úÖ No overlap
Channel 2-5 ‚ùå Overlaps with 1 and 6
Channel 6 ‚úÖ No overlap
Channel 7-10 ‚ùå Overlaps with 6 and 11
Channel 11 ‚úÖ No overlap

‚ö†Ô∏è The Channel Problem

Each 2.4 GHz channel is 22 MHz wide, but they're spaced only 5 MHz apart. This causes massive overlap!

Only channels 1, 6, and 11 are separated by 25 MHz (5 channels) and don't overlap.

Design Rule: In a multi-AP environment, alternate between channels 1, 6, and 11 to minimize co-channel interference.

5 GHz Channel Planning

Much more spectrum available - many non-overlapping channels:

UNII Band Frequency Range Channels Notes
UNII-1 5.150 - 5.250 GHz 36, 40, 44, 48 Indoor only, low power
UNII-2 5.250 - 5.350 GHz 52, 56, 60, 64 DFS required (radar detection)
UNII-2 Extended 5.470 - 5.725 GHz 100, 104, ... 144 DFS required
UNII-3 5.725 - 5.825 GHz 149, 153, 157, 161, 165 Higher power allowed
DFS (Dynamic Frequency Selection): APs must detect radar on UNII-2/2E channels and automatically change channels to avoid interference with weather radar and military systems. This can cause brief disconnections.

5 GHz Advantages

  • 24+ non-overlapping 20 MHz channels
  • Up to 12 non-overlapping 40 MHz channels
  • Up to 6 non-overlapping 80 MHz channels (802.11ac/ax)
  • Less interference from non-Wi-Fi devices

Transmit Power & Coverage

‚ö†Ô∏è Common Misconception

"More power = better coverage" is FALSE!

The Problem with High Power

  • Asymmetric Link: Client hears AP, but AP can't hear client (clients have lower power)
  • Co-channel Interference: High power causes more overlap with other APs
  • Hidden Node: Clients too far apart can't hear each other but both reach AP
  • Roaming Delays: Client stays connected to distant AP too long

‚úÖ Best Practices

  • Use moderate power and more APs for better coverage
  • Overlap cells by 10-15% at the edge (not center)
  • Let WLC automatically adjust power (RRM - Radio Resource Management)
  • Design for signal strength of -67 dBm or better at cell edge
WLC Automatic Power Control: The WLC can automatically adjust AP transmit power based on detected neighbors and interference. This is part of RRM (Radio Resource Management).

üîå Network Interface Card (NIC)

Your device's gateway to the network Layer 2

What It Does

  • Connects a device to the network medium
  • Contains unique MAC address (burned-in)
  • Converts data between parallel (internal) and serial (network)
  • Handles frame creation and error detection

Types of NICs

  • Ethernet: RJ-45, most common
  • Wireless: 802.11 Wi-Fi
  • Fiber: SFP/SFP+ modules

üí° MAC Address

48-bit address written as 6 hex pairs:

AA:BB:CC:DD:EE:FF

First 3 bytes: OUI (Manufacturer)

Last 3 bytes: Unique identifier

üìù Exam Tip

NICs operate at Layer 2 - they use MAC addresses, not IP addresses!

‚≠ï Hub (Legacy Device)

Simple but inefficient Layer 1

How It Works

  • Receives signal on one port
  • Repeats signal to ALL other ports
  • No intelligence - just electrical repeater
  • Creates single collision domain

Types of Hubs

  • Passive: No amplification
  • Active: Regenerates signal
  • Smart: Basic management features

‚ö†Ô∏è Why Hubs Are Obsolete

  • Everyone shares bandwidth
  • Collisions reduce throughput
  • Security risk - everyone sees all traffic
  • Half-duplex only

üìù Exam Tip

Hubs are Layer 1 devices - they don't understand MAC addresses. They just repeat electrical signals.

üåâ Bridge

Segments networks by MAC address Layer 2

What It Does

  • Connects two network segments
  • Learns MAC addresses on each port
  • Forwards frames only when necessary
  • Creates separate collision domains

How It Learns

  • Examines source MAC of incoming frames
  • Builds MAC address table
  • Forwards based on destination MAC
  • Floods unknown destinations

üîÄ Switch

The backbone of modern LANs Layer 2

What It Does

  • Multi-port bridge (many ports, same logic)
  • Creates dedicated collision domain per port
  • Enables full-duplex communication
  • Builds and maintains MAC address table

Switching Methods

  • Store-and-Forward: Buffers entire frame, checks CRC
  • Cut-Through: Forwards after reading dest MAC (faster)
  • Fragment-Free: Reads first 64 bytes (collision check)

üìù Exam Tip

Switches break up collision domains but NOT broadcast domains. All ports are in the same broadcast domain (unless using VLANs).

üîÄ Switch Features & Types

Unmanaged Switch

Plug-and-play, no configuration. Good for home/small office. No VLANs, no monitoring.

Managed Switch

Full configuration via CLI/GUI. VLANs, port security, SNMP monitoring, STP support.

Layer 3 Switch

Switch + Router in one. Routes between VLANs using hardware (ASICs). Very fast!

PoE Switch

Power over Ethernet - powers devices like phones, cameras, APs through the data cable.

üéØ Key Switch Features to Know

  • VLANs: Segment broadcast domains
  • Trunking: Carry multiple VLANs
  • STP: Prevent loops
  • Port Security: Limit MAC addresses
  • Port Mirroring: Copy traffic for analysis
  • Link Aggregation: Bond multiple ports

üåê Router

Connects different networks Layer 3

What It Does

  • Forwards packets between networks
  • Uses IP addresses for decisions
  • Maintains routing table
  • Breaks up broadcast domains

Routing Decisions

  • Examines destination IP
  • Looks up in routing table
  • Determines best next hop
  • Decrements TTL, recalculates checksum

Routers break up both collision domains AND broadcast domains. Each interface is a separate broadcast domain!

üí° When to Use Which

Use Router: Internet edge, WAN connections, branch offices, advanced security

Use L3 Switch: Data center core, high-speed inter-VLAN routing, campus distribution

üî• Firewall

Your network's security guard Layer 3-7

What It Does

  • Filters traffic based on rules
  • Blocks unauthorized access
  • Can inspect packet contents
  • Logs security events

Firewall Types

  • Packet Filter: Basic L3/L4 rules
  • Stateful: Tracks connections
  • Application/NGFW: Deep packet inspection
  • WAF: Web Application Firewall

üõ°Ô∏è IDS / IPS

Intrusion Detection and Prevention Systems

IDS - Intrusion Detection System

  • Monitors traffic passively
  • Alerts on suspicious activity
  • Does NOT block traffic
  • Connected via port mirror/TAP

Think: Security camera - watches and reports

IPS - Intrusion Prevention System

  • Sits inline with traffic
  • Actively blocks threats
  • Can drop malicious packets
  • May introduce latency

Think: Security guard - watches AND intervenes

üîç Detection Methods

Signature-Based:

Matches known attack patterns. Fast, but can't detect new threats.

Anomaly-Based:

Learns "normal" behavior, alerts on deviations. Can detect zero-days but more false positives.

üì∂ Wireless Access Points

Bridging wired and wireless networks Layer 2

What It Does

  • Converts wired to wireless signals
  • Creates wireless coverage area (cell)
  • Bridges wireless clients to wired LAN
  • Handles 802.11 protocols

AP Types

  • Standalone/Autonomous: Self-configured
  • Controller-Based: Centrally managed
  • Cloud-Managed: Configured via cloud

üì° Range Extender / Repeater

Receives and retransmits signal. Extends coverage but halves bandwidth (receives and sends on same channel).

üåê Wireless Bridge

Connects two wired networks over wireless. Used for building-to-building links.

üìù Exam Tip

APs use CSMA/CA (Collision Avoidance), not CSMA/CD like wired Ethernet.

üîÑ DHCP Relay Agent

DHCP broadcasts don't cross routers! A DHCP Relay Agent (ip helper-address on Cisco) forwards DHCP requests to a server on another subnet.

‚öñÔ∏è Load Balancer

Distributing traffic across multiple servers Layer 4-7

What It Does

  • Distributes client requests across servers
  • Provides high availability
  • Performs health checks
  • Enables horizontal scaling

Load Balancing Methods

  • Round Robin: Each server in turn
  • Least Connections: To least busy server
  • IP Hash: Based on client IP (session persistence)
  • Weighted: More traffic to powerful servers

üîÑ Proxy Server

Intermediary between clients and servers Layer 7

What It Does

  • Acts as middleman for requests
  • Caches content for faster access
  • Filters content (block websites)
  • Hides internal IP addresses
  • Logs user activity

Proxy Types

  • Forward Proxy: Clients ‚Üí Proxy ‚Üí Internet
  • Reverse Proxy: Internet ‚Üí Proxy ‚Üí Servers
  • Transparent: No client configuration needed

Forward Proxy Use Cases

  • Content filtering (block social media)
  • Bandwidth savings (caching)
  • Anonymous browsing
  • Access logging/monitoring

Reverse Proxy Use Cases

  • Load balancing
  • SSL termination
  • Web application firewall
  • DDoS protection

VPN Types Supported

  • SSL VPN: Browser-based access

üìã Key Exam Points

Device Layers

  • Layer 1: Hub, Repeater, Media Converter
  • Layer 2: Switch, Bridge, NIC, AP
  • Layer 3: Router, L3 Switch, Firewall
  • Layer 4-7: Load Balancer, Proxy, IDS/IPS

Collision vs Broadcast Domains

  • Hub: 1 collision domain, 1 broadcast domain
  • Switch: 1 collision domain per port, 1 broadcast domain
  • Router: Separate collision AND broadcast domains

Security Devices

  • Firewall: Filters traffic by rules
  • IDS: Detects and alerts (passive)
  • IPS: Detects and blocks (inline)
  • DMZ: Buffer zone for public servers

üìù Remember

Know what layer each device operates at - this is a FAVORITE exam topic!

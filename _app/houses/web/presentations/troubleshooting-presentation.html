üéØ The 7-Step Troubleshooting Methodology

CompTIA's structured approach to solving network problems:

1
Identify
2
Theory
3
Test
4
Plan
5
Implement
6
Verify
7
Document
Memorize this order! Questions often ask "What is the NEXT step?" or "What should you do FIRST?"

üìã Methodology Steps Explained

  • Identify the problem - Gather information, question users, identify symptoms, determine scope (one user or many?)
  • Establish a theory - Question the obvious first, consider multiple approaches, use OSI model (bottom-up, top-down, divide-and-conquer)
  • Test the theory - Confirm or eliminate theories; if not confirmed, establish new theory or escalate
  • Establish a plan of action - Identify potential effects, get approval if needed, schedule maintenance window
  • Implement the solution - Execute the plan, escalate if beyond your expertise
  • Verify full functionality - Test the fix, implement preventive measures
  • Document findings - Record actions, outcomes, and lessons learned

üèì The ping Command

Tests basic connectivity using ICMP Echo Request/Reply

C:\> ping 192.168.1.1

Pinging 192.168.1.1 with 32 bytes of data:
Reply from 192.168.1.1: bytes=32 time=1ms TTL=64
Reply from 192.168.1.1: bytes=32 time<1ms TTL=64
Reply from 192.168.1.1: bytes=32 time<1ms TTL=64
Reply from 192.168.1.1: bytes=32 time<1ms TTL=64

Key Options:

  • -t - Continuous ping (Windows)
  • -n [count] - Number of pings
  • -l [size] - Packet size
  • -a - Resolve to hostname

What It Tells You:

  • Host is reachable (or not)
  • Round-trip time (latency)
  • Packet loss percentage
  • TTL value (hop count hint)

üó∫Ô∏è traceroute / tracert

Maps the path packets take to reach a destination

C:\> tracert google.com

Tracing route to google.com [142.250.80.46]
1 1 ms 1 ms 1 ms 192.168.1.1
2 10 ms 12 ms 11 ms 10.0.0.1
3 15 ms 14 ms 15 ms isp-router.net
4 20 ms 22 ms 21 ms core-router.isp.net
5 25 ms 24 ms 25 ms 142.250.80.46

How It Works:

Sends packets with incrementing TTL values (1, 2, 3...). Each router decrements TTL and sends back "Time Exceeded" when TTL=0, revealing its IP address.

Windows: tracert

Uses ICMP Echo Requests

Linux: traceroute

Uses UDP by default (can use ICMP with -I)

‚öôÔ∏è IP Configuration Commands

View and manage network adapter settings

C:\> ipconfig /all
C:\> ipconfig /release
C:\> ipconfig /renew
C:\> ipconfig /flushdns
C:\> ipconfig /displaydns

Linux: ip / ifconfig

If a Windows PC has a 169.254.x.x address (APIPA), it means DHCP failed. Check DHCP server, network connectivity, or use ipconfig /renew.

üîç DNS Troubleshooting: nslookup & dig

Query DNS servers to resolve names and troubleshoot DNS issues

nslookup (Windows/Linux)

C:\> nslookup google.com
C:\> nslookup -type=MX gmail.com
C:\> nslookup google.com 8.8.8.8

dig (Linux - more detailed)

$ dig google.com
$ dig MX gmail.com
$ dig @8.8.8.8 google.com
$ dig +trace google.com

Common DNS Record Types:

  • A - IPv4 address | AAAA - IPv6 address
  • MX - Mail server | CNAME - Alias
  • PTR - Reverse lookup | NS - Name server

üìã arp - Address Resolution Protocol

View and manage the ARP cache (IP to MAC address mappings)

C:\> arp -a

Interface: 192.168.1.10 --- 0x5
Internet Address Physical Address Type
192.168.1.1 aa-bb-cc-dd-ee-ff dynamic
192.168.1.20 11-22-33-44-55-66 dynamic
192.168.1.255 ff-ff-ff-ff-ff-ff static

Commands:

  • arp -a - Display ARP cache
  • arp -d * - Clear ARP cache
  • arp -s [IP] [MAC] - Add static entry

Troubleshooting Uses:

  • Detect duplicate IP addresses
  • Identify MAC address of devices
  • Diagnose ARP spoofing attacks
Security Note: Duplicate MAC addresses for different IPs may indicate ARP spoofing/poisoning attack!

üìà pathping & mtr - Advanced Path Analysis

Combines ping and traceroute for comprehensive path analysis

Windows: pathping

C:\> pathping google.com

Traces route, then sends 100 pings to each hop to measure packet loss and latency at each point.

Linux: mtr

$ mtr google.com

Real-time, continuously updating display of route and statistics. Combines best of ping and traceroute.

When to Use:

  • Intermittent connectivity issues
  • Identifying which hop has packet loss
  • Latency spikes at specific points in the path
  • ISP or routing problems

üîå Common Wired Network Issues

Problem Symptoms Solution
Duplex Mismatch Slow speeds, late collisions, CRC errors Set both ends to same duplex (preferably auto)
Speed Mismatch Connection drops, very slow transfer Match speed settings on both ends
Bad Cable Intermittent connectivity, high errors Test with cable tester, replace cable
Broadcast Storm Network slowdown, high CPU on switches Enable STP, find and break loop
VLAN Mismatch Can't communicate with expected hosts Verify VLAN assignment on ports
Incorrect Cable Type No link light, no connectivity Use correct cable (straight vs crossover)
Modern switches and NICs use Auto-MDIX to automatically detect cable type, but older equipment may require correct cable selection.

üì∂ Common Wireless Network Issues

Problem Symptoms Solution
Interference Slow speeds, dropped connections Change channel, use 5GHz, identify source
Channel Overlap Poor performance in dense areas Use non-overlapping channels (1, 6, 11)
Insufficient Coverage Dead zones, weak signal areas Add APs, adjust power, use repeaters
Wrong SSID/Password Can't connect, authentication failure Verify credentials, check caps lock
Overcapacity Slow speeds with many users Add APs, load balancing, band steering
Security Mismatch Can't connect to network Match security type (WPA2/WPA3)

2.4 GHz Interference Sources:

Microwaves, cordless phones, Bluetooth devices, baby monitors, neighboring WiFi networks

üî¢ IP Addressing Issues

Common Problems:

  • Duplicate IP - Two devices with same IP cause intermittent issues
  • APIPA Address - 169.254.x.x means DHCP failure
  • Wrong Subnet - Can't reach local resources
  • Wrong Gateway - Can reach local but not internet
  • Exhausted DHCP Pool - New devices can't get addresses

Diagnostic Steps:

  • Check IP with ipconfig / ip addr
  • Ping loopback (127.0.0.1)
  • Ping own IP address
  • Ping default gateway
  • Ping remote host
  • Test DNS resolution
APIPA (169.254.x.x): Windows auto-assigns this when DHCP is unavailable. Check: DHCP server running? Network cable connected? Correct VLAN?

üåê DNS Troubleshooting

When name resolution fails

Symptoms:

  • Can ping IP but not hostname
  • "Server not found" errors
  • Website loads by IP but not name
  • Slow browsing (DNS timeout)

Solutions:

  • Flush DNS cache: ipconfig /flushdns
  • Try different DNS server (8.8.8.8)
  • Check DNS server settings
  • Verify DNS server is reachable
C:\> nslookup example.com 8.8.8.8
Server: dns.google
Address: 8.8.8.8

Non-authoritative answer:
Name: example.com
Address: 93.184.216.34
If nslookup works with 8.8.8.8 but not the default DNS, the issue is with the configured DNS server, not the network.

‚ö° Network Performance Issues

Issue Cause Solution
High Latency Distance, congestion, processing delays Optimize routing, upgrade links, QoS
Jitter Variable latency, affects VoIP/video QoS prioritization, jitter buffer
Packet Loss Congestion, bad cables, interference Identify bottleneck, fix physical issues
Bandwidth Saturation Too much traffic for link capacity Upgrade link, traffic shaping, QoS

Key Metrics:

  • Latency: Time for packet to travel (measured in ms)
  • Jitter: Variation in latency
  • Throughput: Actual data transfer rate
  • Bandwidth: Maximum theoretical capacity

üéØ OSI-Based Troubleshooting Approaches

Bottom-Up (L1 ‚Üí L7)

  • Start with physical layer
  • Check cables, lights, power
  • Move up through each layer
  • Best for: Hardware issues, new installations

Top-Down (L7 ‚Üí L1)

  • Start with application
  • Test if app works, then services
  • Move down through layers
  • Best for: Application-specific issues

Divide and Conquer

Start in the middle (Layer 3/4), then go up or down based on results. If ping works (L3), issue is above; if not, check below.

Quick Test Sequence:

1. Ping 127.0.0.1 (loopback) ‚Üí 2. Ping own IP ‚Üí 3. Ping gateway ‚Üí 4. Ping remote IP ‚Üí 5. Ping hostname

Cloud Concepts

Service Models

🖥

IaaS - Infrastructure as a Service

  • Virtual machines, storage, networks
  • You manage: OS, apps, data
  • Example: Azure Virtual Machines
🛠

PaaS - Platform as a Service

  • Development framework, databases
  • You manage: apps, data
  • Example: Azure App Service, Azure SQL
💻

SaaS - Software as a Service

  • Complete applications via browser
  • You manage: your data only
  • Example: Microsoft 365, Dynamics 365

Shared Responsibility Model

Component IaaS PaaS SaaS
Data You You Shared
Applications You You Microsoft
Runtime You Microsoft Microsoft
OS You Microsoft Microsoft
Infrastructure Microsoft Microsoft Microsoft
Exam Tip: You are ALWAYS responsible for your data, accounts, and access management - regardless of service model!

Azure Architecture & Geography

Azure Geography
Data residency boundary (e.g., United States, Europe)
↓
Region
Physical datacenter location (e.g., East US, West Europe)
↓
Availability Zone
Isolated datacenter within a region (AZ 1, 2, 3)
↓
Resource Groups
Logical container for Azure resources

Key Concepts

Region: Set of datacenters deployed within a latency-defined perimeter, connected by a dedicated regional network. 60+ regions worldwide.
Availability Zone: Physically separate locations within an Azure region. Each has independent power, cooling, and networking. Protects from datacenter failures.

Management Hierarchy

Management Groups - Organize subscriptions
  ↓
Subscriptions - Billing boundary
  ↓
Resource Groups - Logical grouping
  ↓
Resources - VMs, databases, storage

Azure Storage Services

Storage Account Types

Blob Storage - Object storage for unstructured data
  • Images, videos, backups, logs
  • Hot, Cool, Archive tiers
  • Block, Page, Append blobs
File Storage - Managed file shares (SMB/NFS)
  • Replace on-prem file servers
  • Mount from Windows, Linux, macOS
  • Premium and Standard tiers
Queue Storage - Message queue service
  • Store millions of messages
  • Decouple application components
  • Asynchronous processing
Table Storage - NoSQL key-value store
  • Schema-less design
  • Structured non-relational data
  • Fast and cost-effective

Storage Tiers (Blob)

Hot Tier

Use: Frequently accessed data

Cost: High storage, low access

Example: Active website images

Cool Tier

Use: Infrequently accessed (30+ days)

Cost: Lower storage, higher access

Example: Short-term backups

Archive Tier

Use: Rarely accessed (180+ days)

Cost: Lowest storage, highest access

Example: Compliance archives

Redundancy Options

LRS: Locally Redundant (3 copies, same datacenter)
ZRS: Zone Redundant (3 zones, same region)
GRS: Geo-Redundant (secondary region)
GZRS: Geo-Zone Redundant (zones + geo)

Azure Networking

Virtual Network (VNet)

What: Isolated network in Azure
  • Define IP address space (CIDR)
  • Create subnets
  • Connect VNets via peering
  • Route tables and NSGs for security

Load Balancer

Layer 4 (TCP/UDP) load balancing
  • Distribute traffic across VMs
  • Health probes for availability
  • Public and Internal LB options

Application Gateway

Layer 7 (HTTP/HTTPS) load balancer
  • URL-based routing
  • SSL termination
  • Web Application Firewall (WAF)

VPN Gateway

What: Encrypted connection over Internet
  • Site-to-Site: On-prem to Azure
  • Point-to-Site: Individual device to Azure
  • VNet-to-VNet: Azure VNet to VNet

ExpressRoute

What: Private dedicated connection
  • Does NOT go over public Internet
  • Higher reliability and speed
  • Consistent latency
  • More expensive than VPN

VPN vs ExpressRoute

VPN: Driving on public roads with encryption
ExpressRoute: Your own private highway to Azure

Exam Tip: ExpressRoute is private and does NOT traverse the Internet. VPN uses public Internet with encryption.

Azure Identity - Azure Active Directory

Azure AD (Azure Entra ID)

Cloud-based identity service
  • User and group management
  • Single Sign-On (SSO) to thousands of apps
  • NOT the same as Windows Server AD
  • Supports B2B (business) and B2C (consumer)

Multi-Factor Authentication (MFA)

Extra layer of security
  • Something you know (password)
  • Something you have (phone, token)
  • Something you are (biometric)

Conditional Access

Policy-based access control
  • Require MFA for specific apps
  • Block access from certain locations
  • Require compliant devices

Role-Based Access Control (RBAC)

Grant least-privilege access
  • Owner: Full control including access management
  • Contributor: Create/manage resources, no access control
  • Reader: View resources only
  • Custom roles: Specific permissions

RBAC Scope Levels

  • Management Group: Multiple subscriptions
  • Subscription: All resources in subscription
  • Resource Group: All resources in group
  • Resource: Specific resource only

Permissions INHERIT down the hierarchy!

Single Sign-On (SSO)

One login for multiple applications
  • Integrated with Azure AD
  • Supports SAML, OAuth, OpenID Connect

Azure Security

🔒

Azure Security Center (Defender for Cloud)

  • Unified security management
  • Threat protection across hybrid cloud
  • Secure Score recommendations
  • Compliance dashboards
🔑

Azure Key Vault

  • Securely store secrets, keys, certificates
  • Hardware Security Module (HSM) backed
  • Centralized secrets management
  • Access logging and monitoring
🛡

Azure DDoS Protection

  • Basic: Free, automatic protection
  • Standard: Advanced mitigation, cost protection
  • Always-on traffic monitoring
  • Application-specific tuning

Azure Firewall

Managed, cloud-based network security
  • Stateful firewall as a service
  • Application and network-level filtering
  • Threat intelligence-based filtering
  • High availability built-in

Network Security Groups (NSG)

Filter network traffic to/from Azure resources
  • Inbound and outbound rules
  • Allow or Deny based on 5-tuple (source IP, source port, dest IP, dest port, protocol)
  • Applied to subnets or NICs

Azure Sentinel

Cloud-native SIEM and SOAR
  • Collect data across enterprise
  • AI-powered threat detection
  • Automated threat response
  • Investigation and hunting tools

Defense in Depth

Layered security approach:
  1. Physical: Datacenter security
  2. Identity: Azure AD, MFA
  3. Perimeter: DDoS, Firewall
  4. Network: NSG, segmentation
  5. Compute: Secure VMs, patching
  6. Application: Secure coding
  7. Data: Encryption at rest/transit

Azure Governance & Compliance

Azure Policy

Enforce organizational standards
  • Define rules for resource properties
  • Audit non-compliant resources
  • Prevent non-compliant resources from being created
  • Auto-remediation capabilities

Example: Require all VMs to have backup enabled, or only allow specific VM sizes.

Azure Blueprints

Repeatable environment setup
  • Package of artifacts (policies, RBAC, ARM templates)
  • Version control for governance
  • Deploy compliant environments quickly
  • Relationship between blueprint and deployment is preserved

Resource Locks

Prevent accidental deletion/modification
  • CanNotDelete: Can modify, can't delete
  • ReadOnly: Can't modify or delete
  • Applied to subscriptions, resource groups, or resources

Management Groups

Organize multiple subscriptions
  • Apply policies across multiple subscriptions
  • Hierarchical structure (up to 6 levels)
  • RBAC inheritance
  • Enterprise-scale governance
Root Management Group
↓
Production MG
Development MG
↓
Subscription 1
Subscription 2
Subscription 3

Azure Tags

Organize and categorize resources
  • Key-value pairs (e.g., Department: IT)
  • Use for billing, automation, organization
  • Applied to resources and resource groups
  • Do NOT inherit by default
Exam Tip: Policies ENFORCE compliance (prevent non-compliant resources), Blueprints PACKAGE multiple governance artifacts together

Azure Pricing & Cost Management

Pricing Factors

What affects Azure costs?
  • Resource Type: VMs, storage, bandwidth
  • Consumption: Pay-per-use vs reserved
  • Maintenance: Managed services cost more upfront
  • Geography: Region pricing varies
  • Network Traffic: Inbound free, outbound charged
  • Subscription Type: Free, pay-as-you-go, enterprise

Azure Pricing Calculator

Estimate costs BEFORE deploying
  • Select services and configure options
  • See monthly cost estimates
  • Compare different configurations
  • Export and share estimates

URL: azure.microsoft.com/pricing/calculator

Total Cost of Ownership (TCO) Calculator

Compare on-premises vs Azure costs
  • Enter current infrastructure details
  • Include power, cooling, IT labor
  • See potential savings over 5 years
  • Helps justify cloud migration

Azure Cost Management

Monitor and optimize spending
  • Cost Analysis: View current and historical costs
  • Budgets: Set spending limits and alerts
  • Recommendations: Cost-saving suggestions
  • Exports: Automate cost data exports

Cost Optimization Tips

  • Reserved Instances: 1 or 3-year commitment saves up to 72%
  • Azure Hybrid Benefit: Use existing Windows/SQL licenses
  • Right-sizing: Match VM size to workload
  • Auto-shutdown: Turn off dev/test VMs when not in use
  • Cool/Archive storage: For infrequent access

Azure Service Level Agreements (SLAs)

What is an SLA?

Microsoft's commitment to uptime and connectivity
  • Expressed as percentage (e.g., 99.9%)
  • Different services have different SLAs
  • Service credits if SLA not met

Common Azure SLAs

Service SLA Downtime/Year
VMs (2+ instances in Availability Set) 99.95% ~4.4 hours
VMs (2+ zones) 99.99% ~52 minutes
Azure SQL Database 99.99% ~52 minutes
Azure Storage (LRS) 99.9% ~8.8 hours
App Service 99.95% ~4.4 hours
Important: Free tier and preview services typically have NO SLA!

Composite SLAs

Multiple services = MULTIPLY SLAs

Example: Web App (99.95%) + SQL Database (99.99%)

0.9995 0.9999 = 0.9994 = 99.94%

More dependencies = Lower overall SLA!

Improving SLAs

Strategies:
  • Availability Zones: Deploy across zones (99.99%)
  • Availability Sets: Multiple VMs in same datacenter (99.95%)
  • Redundancy: Duplicate critical components
  • Load Balancing: Distribute traffic
  • Backup/DR: Regional pairs for disaster recovery

Application SLA

Your SLA to customers
  • Should be lower than composite Azure SLA
  • Build in buffer for maintenance, updates
  • Design for redundancy and failover

Key Exam Topics

Cloud Concepts (25-30%)

  • Deployment models: Public, Private, Hybrid
  • Shared responsibility model
Azure Architecture (35-40%)
  • Regions, Availability Zones, Resource Groups
  • Compute: VMs, App Service, Functions, Containers
  • Storage: Blob, File, Queue, Table, tiers
  • Networking: VNet, VPN, ExpressRoute, Load Balancer
Management & Governance (30-35%)
  • Cost Management, pricing calculators, TCO
  • Azure Policy, Blueprints, RBAC, Locks
  • Azure Monitor, Log Analytics, App Insights
  • Azure Advisor, Service Health

Exam Success Tips

1. HANDS-ON PRACTICE: Create a free Azure account and practice! ($200 free credit for 30 days)
2. Know the DIFFERENCE between similar services (VPN vs ExpressRoute, Policy vs Blueprints, etc.)
3. Understand SHARED RESPONSIBILITY - what YOU manage vs what MICROSOFT manages
4. Memorize common SLAs and how to calculate composite SLAs
5. Use Microsoft Learn modules - they're FREE and exam-aligned!

Common Exam Question Types

  • Multiple Choice: Single correct answer
  • Multiple Response: Select all that apply
  • Drag and Drop: Match items or order steps
  • Case Studies: Real-world scenarios (3-5 questions each)
  • Review Screen: Mark questions for review before submitting

Final Reminders

Always answer based on AZURE best practices, not general cloud concepts

When in doubt, choose the MOST SECURE or MOST AVAILABLE option

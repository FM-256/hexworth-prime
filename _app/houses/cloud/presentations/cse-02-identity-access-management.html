🔄 Legacy vs. Modern IAM

Legacy IAM

  • Installed on organization's local infrastructure
  • Lacks unified, centralized architecture
  • Does not support Single Sign-On (SSO)
  • Decentralized authentication processes
  • Users added to individual application databases

Modern IAM

  • Leverages cloud identity solutions
  • Enables seamless access with SSO
  • Hybrid architecture with on-prem AD DS
  • Enhanced security through MFA
  • Self-service Password Reset (SSPR)

🛠 Role-Based Access Control (RBAC)

RBAC ensures precise cloud access by defining roles, permissions, and linking users in IAM. It broadens access control beyond cloud IAM, fostering unified, role-based security across systems.

Owner/Root
Full control across resources
Contributor
Create specific resources
Reader
View and generate reports
Guest
Limited access permissions

🔀 Identity Federation

Identity federation establishes a single security platform by seamlessly integrating and unifying authentication and access management across on-premises and cloud environments.

Hybrid Federation
Syncs cloud and on-prem AD
Multicloud Federation
Single IdP for multiple clouds
B2B Federation
Partner-to-partner access
B2C Federation
Consumer login via external IdP

🔓 Single Sign-On (SSO)

Users sign in once and the identity provider holds their "token" for a predetermined session time (typically 24 hours). When the session expires, the user must sign in again.

🔐 Self-Service Password Reset (SSPR)

Users verify their identity before resetting passwords through methods like one-time passcodes or authenticator apps. SSPR enhances security against social engineering and reduces support calls.

🔐 Multifactor Authentication (MFA)

🧠
Something You Know
Knowledge-based factors like a PIN or password
📱
Something You Have
Possession-based factors like a mobile device or security token
👀
Something You Are
Inherence-based factors like fingerprints or facial recognition

🔒 Principle of Least Privilege

Just In Time Access

Providing users with access rights only when needed and for the minimum duration necessary to perform a specific task.

Just Enough Access

Granting users the minimum level of access or permissions required to perform their specific tasks or responsibilities.

🛡 Conditional & Behavior-Based Access

Conditions encompass criteria assessed before allowing access. Controls determine what actions users can perform once access is granted based on specified conditions.

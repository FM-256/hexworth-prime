Fundamental principles for cloud application design

✅
Reliability
The software functions as expected
💪
Resiliency
Software and hardware can withstand misuse or attack
🔄
Recovery
Application can be restored with minimal disruption

Security Controls within CIA

Protecting Confidentiality, Integrity, and Availability

🔒
Confidentiality
  • > Avoid exposure of application data
  • > Minimize sensitive data
  • > Mask data with steganography
  • > Encrypt and control access
🛡
Integrity
  • > Avoid tampering of data
  • > Use secure hashing techniques
  • > Digital signatures for auth
  • > Separation of duties
✅
Availability
  • > Data accessible when needed
  • > Redundancy and fault tolerance
  • > Guard against DDoS attacks
  • > Build DR in the cloud

🛡 Web Application Firewall (WAF)

Protects cloud networks from security breaches at the application level. Filters and monitors web traffic, defends against XSS, CSRF, SQL injection, and brute force attacks.

Azure: Application Gateway + Front Door integration | AWS: WAF with ALB + CloudFront

🔍 Secure Code Testing

SAST (White Box)

  • Analyzes source code
  • Early detection of misconfigurations
  • Works with any application code
  • No production vulnerabilities detected
  • Identifies issues before deployment

DAST (Black Box)

  • Tests active applications
  • Uses live production data
  • Identifies runtime vulnerabilities
  • Primarily for web applications
  • Finds post-production issues

⚡ Serverless Security

  • One role per function (least privilege)
  • Use secrets manager - no secrets in code
  • Know your dependencies (vulnerability scans)
  • Use variables for configs
  • Tighten access controls
  • Automate security in CI/CD
  • Clean up /tmp directories

📦 Container Security

  • Secure your images
  • Manage secrets securely
  • Restrict container privileges at runtime
  • Fix security misconfigurations
  • Automate vulnerability scanning
  • Real-time logging and alerting
  • Adopt "shift left" security

🔗 API Security Best Practices

Document security requirements using NIST and OWASP resources. Include logic in design reviews. Draft secure coding practices. Use API Management to hide backend URLs, apply rate limiting, and monitor through APIM.

💡 Key Takeaways

✓ 3 Rs: Reliability, Resiliency, Recovery
✓ OWASP Top 10 defines critical web vulnerabilities
✓ WAFs protect against XSS, SQLi, and code injection
✓ SAST tests code; DAST tests running applications
✓ Serverless: one role per function, no secrets in code
✓ Containers: secure images, restrict runtime privileges

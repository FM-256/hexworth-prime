🖧 Cloud Networks

Cloud networks consist of virtual connections and interfaces instead of physical ports and patch cables. They utilize private IP address schema for customer resources, with external connections provided via public IP addresses.

Cloud networks are regional resources - all resources using the network must be in that region. Networks can be "peered" to extend across regions on the cloud provider backbone.

🛡 NACLs vs Security Groups

Network ACLs (NACL)

  • Stateless - separate inbound/outbound rules
  • Each subnet associates with one NACL
  • One NACL can serve multiple subnets
  • Can block specific IP addresses
  • Default NACL allows all traffic
  • Custom NACL denies all by default

Security Groups

  • Stateful - combined in/out rules
  • Deny rule applies to both directions
  • Block ports, not IP addresses
  • Applied at instance/NIC level
  • Azure NSGs: 3 default in + 3 out rules
  • Low-cost security solution

Remote Access & Connections

Secure connectivity to cloud resources

🔀

VPC Endpoints

Internal gateway for secure AWS resource access. Assigns elastic network interface for private connectivity without internet exposure.

🔄

Transit Gateway

Centralized hub connecting thousands of VPCs and on-premises resources. Scalable hub-and-spoke model with route tables.

🔗

Private Links

Direct secure connections to Azure services without public internet. Enables on-prem to cloud connectivity via Microsoft backbone.

💻

Azure Bastion

Secure RDP/SSH without exposing management ports. Access via Azure Portal with no public IP required on VMs.

⏰

Just-In-Time VM Access

On-demand, time-limited access to VMs. Minimizes attack surface by closing ports when not in use.

🖧

NAT Gateway

Preferred over NAT instances for private subnet internet access. HA across availability zones, scalable and burstable.

🔥 Firewalls in the Cloud

🛡 Azure Firewall

  • Stateful Firewall as a Service
  • High availability and zone redundancy
  • Next-generation firewall features
  • Packet filtering by attributes

🌐 Web Application Firewall

  • Protects against XSS attacks
  • Guards against SQL injection
  • Prevents code injection
  • Works with Application Load Balancer
  • Integrates with CloudFront/Front Door

👁 IDS vs IPS

Intrusion Detection System

  • Monitors and detects potential threats
  • Non-intrusive - doesn't alter traffic
  • Notifies without modifying packets
  • Enables rapid incident response
  • Passive observation mode

Intrusion Prevention System

  • Monitors and detects like IDS
  • Acts preemptively to block threats
  • Blocks or alters malicious traffic
  • Safeguards network integrity
  • Active protection mode

💡 Key Takeaways

✓ Cloud networks are virtual - no physical cables or ports
✓ VPCs provide logical isolation within cloud providers
✓ NACLs are stateless; Security Groups are stateful
✓ Use Private Links/Endpoints for secure service access
✓ JIT access minimizes exposure of management ports
✓ IDS detects; IPS prevents - use both for defense in depth

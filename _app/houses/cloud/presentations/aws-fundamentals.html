AWS Account Structure

Account Hierarchy

👑

Root User

  • Created when you sign up for AWS
  • Full access to all AWS services
  • Email address + password
  • NEVER use for daily tasks!
👤

IAM Users

  • Individual identities with credentials
  • Specific permissions assigned
  • Use for daily operations
  • Can have MFA enabled

AWS Organizations

Centrally manage multiple AWS accounts

Features:
  • Consolidated Billing: One bill for all accounts
  • Service Control Policies (SCPs): Restrict actions across accounts
  • Organizational Units (OUs): Group accounts logically (e.g., Dev, Test, Prod)
  • Volume Discounts: Combined usage = better pricing

Root User Best Practices

Critical Security Steps:
  • Enable MFA immediately
  • Delete root access keys
  • Use only for account setup
  • Create IAM users for all work

Amazon EC2 (Elastic Compute Cloud)

Instance Types:
  • T3/T4g: General purpose, burstable (web servers)
  • C6i: Compute optimized (batch processing)
  • R6i: Memory optimized (databases)
  • P4: GPU instances (ML/AI)
  • I4i: Storage optimized (big data)
AMI (Amazon Machine Image):
Pre-configured template for your EC2 instance (OS + software)

AWS Lambda

Serverless Computing:
  • Pay only for compute time used
  • Automatically scales
  • No server management
  • Triggered by events (S3, API Gateway, etc.)
  • Max 15 minutes execution time

ECS vs EKS

  • ECS: AWS-native container orchestration
  • EKS: Managed Kubernetes service

Core Storage Services

📦

Amazon S3

Simple Storage Service - Object storage for any amount of data

S3 Storage Classes

Class Use Case
S3 Standard Frequently accessed data
S3 Intelligent-Tiering Unknown/changing access patterns
S3 Standard-IA Infrequent access, quick retrieval
S3 One Zone-IA Infrequent, recreatable data
S3 Glacier Instant Archive, millisecond retrieval
S3 Glacier Flexible Archive, minutes-hours retrieval
S3 Glacier Deep Archive Long-term archive, 12 hours

Other Storage Services

EBS (Elastic Block Store):
  • Block storage for EC2 instances
  • Like a virtual hard drive
  • Single AZ attachment
  • Persistent storage
  • Types: gp3 (general), io2 (high performance)
EFS (Elastic File System):
  • Managed NFS file system
  • Multiple EC2 instances can access
  • Linux-based workloads
  • Auto-scales capacity
S3 vs EBS vs EFS:
S3: Objects (files, media)
EBS: Block storage for ONE EC2
EFS: Shared file system for MANY EC2s

Data Warehouse:

Security & IAM

AWS Identity and Access Management (IAM)

Control access to AWS services and resources securely

👤

Users

Individual people or applications

👥

Groups

Collection of users with shared permissions

👔

Roles

Temporary credentials for services/users

📝

Policies

JSON documents defining permissions

IAM Policy Structure

Components:
  • Effect: Allow or Deny
  • Action: What can be done (e.g., s3:GetObject)
  • Resource: Which AWS resources
  • Condition: When the policy applies (optional)

Best Practices

  • Least Privilege: Grant minimum permissions needed
  • Use Groups: Assign permissions to groups, not users
  • Enable MFA: Especially for privileged users
  • Rotate Credentials: Change passwords/keys regularly

Multi-Factor Authentication (MFA)

Additional security layer:
  • Something you know (password)
  • Something you have (MFA device)
  • Virtual MFA: Google Authenticator, Authy
  • Hardware MFA: YubiKey, Gemalto

IAM Roles Use Cases

  • EC2 instances accessing S3
  • Lambda functions calling other services
  • Cross-account access
  • Temporary access for users
Never: Embed access keys in code or share credentials!

Amazon CloudWatch

Observability service:
  • Metrics: CPU, memory, disk usage
  • Logs: Centralized log management
  • Alarms: Trigger actions on thresholds
  • Dashboards: Visualize metrics
  • Events: Respond to state changes

Common CloudWatch Metrics

  • EC2: CPUUtilization, NetworkIn/Out
  • EBS: VolumeReadOps, VolumeWriteOps
  • RDS: DatabaseConnections, FreeStorageSpace
  • Lambda: Invocations, Duration, Errors

AWS CloudTrail

Audit and compliance:
  • Records all API calls in your account
  • WHO did WHAT, WHEN, WHERE
  • Security analysis and troubleshooting
  • Compliance auditing
  • Stores logs in S3

AWS Config

Configuration management:
  • Track resource configurations over time
  • Evaluate compliance against rules
  • Configuration history and change tracking
  • Relationship tracking (what's connected)
  • Automated remediation

Remember the Difference

  • CloudWatch: Performance monitoring (how healthy is my system?)
  • CloudTrail: Activity logging (who did what?)
  • Config: Configuration tracking (what changed?)

AWS CloudFormation

Infrastructure as Code (IaC) - Define AWS resources using templates

Key Features:
  • Templates written in JSON or YAML
  • Version control your infrastructure
  • Create/update/delete entire stacks
  • Rollback on errors
  • Free service (pay only for resources created)

CloudFormation Benefits

  • Consistency: Same infrastructure every time
  • Repeatability: Deploy to multiple regions/accounts
  • Documentation: Template is the documentation
  • Cost Estimation: Preview costs before deployment
🚀

Platform as a Service (PaaS) - Deploy and manage applications without infrastructure management

Supported Platforms:
  • Java, .NET, PHP, Node.js, Python, Ruby, Go
  • Docker containers
  • Auto-scaling and load balancing included
  • Monitoring with CloudWatch

AWS Systems Manager

Operational management:
  • Patch management (automated updates)
  • Run commands on multiple instances
  • Session Manager (SSH replacement)
  • Parameter Store (secure secrets storage)
  • Automation runbooks

AWS Pricing Models

Model Description Savings Best For
On-Demand Pay per hour/second with no commitments 0% (baseline) Short-term, unpredictable workloads
Reserved Instances 1 or 3-year commitment for specific instance type Up to 75% Steady-state, predictable workloads
Savings Plans Commit to consistent usage ($/hour) for 1 or 3 years Up to 72% Flexible compute needs
Spot Instances Bid on unused EC2 capacity (can be interrupted) Up to 90% Fault-tolerant, flexible workloads
Dedicated Hosts Physical server dedicated to you Variable Compliance, licensing requirements

Cost Optimization Tips

  • Right-size your instances (don't over-provision)
  • Use Auto Scaling to match demand
  • Choose appropriate storage classes
  • Delete unused resources
  • Use S3 lifecycle policies
  • Monitor with Cost Explorer

AWS Free Tier

Three types:
  • Always Free: DynamoDB 25GB, Lambda 1M requests/month
  • 12 Months Free: 750 hours EC2 t2.micro/month, 5GB S3
  • Trials: Short-term offers for specific services
Billing Alerts: Set up CloudWatch billing alarms to avoid surprises!

AWS Well-Architected Framework

A set of best practices for building secure, high-performing, resilient, and efficient infrastructure for applications

1. Operational Excellence

Run and monitor systems to deliver business value

  • Automate processes
  • Document operations
  • Anticipate failure
  • Learn from operational events

2. Security

Protect information and systems

  • Implement strong identity controls
  • Enable traceability (CloudTrail)
  • Apply security at all layers
  • Automate security best practices
  • Encrypt data in transit and at rest

3. Reliability

Recover from failures and meet demand

  • Test recovery procedures
  • Automatically recover from failure
  • Scale horizontally
  • Use multiple Availability Zones
  • Manage change through automation

4. Performance Efficiency

Use computing resources efficiently

  • Use advanced technologies (serverless, containers)
  • Deploy globally in minutes
  • Use serverless architectures
  • Experiment frequently

5. Cost Optimization

Avoid unnecessary costs

  • Adopt consumption-based pricing
  • Measure overall efficiency
  • Analyze spending over time
  • Right-size resources

6. Sustainability

Minimize environmental impact

  • Maximize utilization
  • Use managed services
  • Reduce downstream impact
  • Choose efficient regions

Summary & Exam Tips

Cloud Practitioner Exam (CLF-C02) Overview

  • Duration: 90 minutes
  • Format: 65 questions (multiple choice & multiple response)
  • Passing Score: 700/1000
  • Cost: $100 USD

Exam Domains

  • Cloud Concepts (24%): What is cloud? Benefits, deployment models
  • Security & Compliance (30%): IAM, shared responsibility
  • Cloud Technology & Services (34%): Compute, storage, database, networking
  • Billing, Pricing, & Support (12%): Pricing models, cost optimization

Key Topics to Master

  • Global infrastructure (regions, AZs, edge locations)
  • IAM (users, groups, roles, policies)
  • EC2 instance types and pricing
  • S3 storage classes
  • VPC components (subnets, security groups, NACLs)
  • Shared responsibility model
  • Well-Architected Framework pillars

Exam Success Tips

  • Understand the "why": Not just what services do, but when to use them
  • Know the differences: S3 vs EBS vs EFS, Security Groups vs NACLs
  • Focus on use cases: Match services to scenarios
  • Shared responsibility: Know what AWS manages vs what you manage
  • Pricing models: When to use On-Demand vs Reserved vs Spot
  • Practice exams: Take multiple practice tests before the real exam

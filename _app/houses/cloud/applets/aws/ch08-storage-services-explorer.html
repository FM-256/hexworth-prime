Key Differences Summary

When to Use S3
Choose S3 when you need to store files that don't change frequently, need to share data across multiple applications, or want unlimited scalability. Perfect for backups, media hosting, and data lakes.
When to Use EBS
Choose EBS when you need low-latency access to data from a single EC2 instance. Ideal for databases, boot volumes, and applications requiring consistent IOPS performance.
When to Use EFS
Choose EFS when multiple EC2 instances need to access the same file system simultaneously. Great for content management, web serving, and shared development environments.
When to Use Storage Gateway
Choose Storage Gateway when you need to integrate on-premises applications with cloud storage, migrate data to AWS, or implement hybrid cloud backup solutions.

Key Features:

  • Snapshots: Point-in-time backups stored in S3
  • Encryption: AES-256 encryption with KMS integration
  • Multi-Attach: Attach volume to multiple EC2 instances (io1/io2)
  • Elastic Volumes: Modify size, performance, and type without downtime
  • Fast Snapshot Restore: Instantly restore large snapshots
EBS vs Instance Store
EBS: Persistent storage that survives instance stops/starts. Data is retained even if instance is terminated (if "Delete on Termination" is disabled).

Instance Store: Ephemeral storage physically attached to the host. Data is lost when instance stops or terminates. Faster performance but no persistence.

AWS Storage Gateway

Storage Gateway Types

File Gateway

NFS/SMB interface to S3. Store files as objects in S3 with local caching.

Volume Gateway

iSCSI block storage backed by S3. Cached or stored volume modes.

Tape Gateway

Virtual tape library backed by S3 and Glacier for backup applications.

Use Cases:

  • Cloud Migration: Gradually move on-premises data to AWS
  • Backup & DR: Use AWS as backup target for on-premises data
  • Tiered Storage: Keep hot data on-premises, cold data in S3
  • Hybrid Workflows: Seamlessly integrate cloud and on-premises storage

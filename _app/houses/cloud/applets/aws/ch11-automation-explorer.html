AWS CloudFormation
Infrastructure as Code
AWS CloudFormation provides a common language to model and provision AWS and third-party resources in your cloud environment. You create templates that describe AWS resources, and CloudFormation handles provisioning and configuring them.
Key Features
  • Infrastructure as Code using JSON or YAML templates
  • Supports all AWS resources and many third-party resources
  • Automatic dependency management and resource ordering
  • Stack updates with change sets for preview
  • Rollback on failure for safe deployments
  • Cross-region and cross-account deployments
  • Drift detection to identify manual changes
  • StackSets for multi-account/region deployments
Best Use Cases
  • Provisioning entire application stacks
  • Replicating infrastructure across environments
  • Version control for infrastructure
  • Disaster recovery and backup strategies
  • Compliance and governance enforcement
Pricing
  • No additional charge for CloudFormation
  • Pay only for AWS resources created
  • Free tier includes unlimited templates and stacks

AWS Elastic Beanstalk
Platform as a Service
AWS Elastic Beanstalk is an easy-to-use service for deploying and scaling web applications. You simply upload your code, and Elastic Beanstalk automatically handles deployment, capacity provisioning, load balancing, and auto-scaling.
Key Features
  • Supports multiple platforms (Node.js, Python, Ruby, Java, .NET, PHP, Go, Docker)
  • Automatic capacity provisioning and load balancing
  • Built-in application health monitoring
  • Managed platform updates and patching
  • Environment cloning for easy replication
  • Blue/green deployments for zero downtime
  • Full control over underlying resources
  • Integration with CloudWatch, X-Ray, and RDS
Best Use Cases
  • Web applications and APIs
  • Applications with standard architectures
  • Rapid application deployment and iteration
  • Developer-friendly PaaS experience
  • Applications requiring automatic scaling
Pricing
  • No additional charge for Elastic Beanstalk
  • Pay only for underlying AWS resources (EC2, S3, RDS, etc.)
  • Free tier eligible for small applications

AWS OpsWorks
Configuration Management
AWS OpsWorks is a configuration management service that provides managed instances of Chef and Puppet. It helps automate server configuration, deployment, and management at scale.
Key Features
  • Chef and Puppet automation platforms
  • OpsWorks Stacks for application lifecycle management
  • Layer-based architecture (web, app, database)
  • Lifecycle events and hooks
  • Auto-healing for failed instances
  • Time-based and load-based auto-scaling
  • Integration with CloudWatch and CloudTrail
  • Recipe-based configuration management
Best Use Cases
  • Complex application configurations
  • Existing Chef or Puppet workflows
  • Multi-tier applications with dependencies
  • Configuration management at scale
  • Hybrid cloud deployments
Pricing
  • OpsWorks Stacks: No additional charge
  • OpsWorks for Chef/Puppet: Charged per node per hour
  • Pay for underlying AWS resources

AWS Systems Manager
Operations Management
AWS Systems Manager provides unified interface to view and control your AWS infrastructure. It helps automate operational tasks across AWS resources and provides visibility into resource compliance.
Key Features
  • Run Command for remote execution
  • Session Manager for secure instance access
  • Patch Manager for automated patching
  • State Manager for configuration compliance
  • Parameter Store for configuration data
  • Automation for common maintenance tasks
  • OpsCenter for operational issue management
  • Inventory for metadata collection
Best Use Cases
  • Fleet management and patching
  • Configuration and compliance management
  • Operational task automation
  • Secure remote access without SSH keys
  • Centralized parameter and secret management
Pricing
  • Many features are free (Run Command, Session Manager)
  • Some features have tiered pricing (Parameter Store, OpsCenter)
  • Pay-per-use for on-premises instances

Choosing the Right Tool
1
Use CloudFormation When:
You need to provision and manage any type of AWS resource with full control. Ideal for complex architectures, multi-tier applications, and teams practicing Infrastructure as Code. Best when you need version control, change management, and repeatable infrastructure deployments.
2
Use Elastic Beanstalk When:
You want to focus on application code without managing infrastructure. Perfect for developers who need to quickly deploy web applications or APIs. Ideal for standard application architectures with common patterns (load balancing, auto-scaling, monitoring).
3
Use OpsWorks When:
You have existing Chef or Puppet expertise and need configuration management at scale. Best for complex application configurations with multiple dependencies. Useful when migrating from on-premises environments with existing automation.
4
Combine Multiple Tools:
Many organizations use multiple tools together. For example, use CloudFormation to provision base infrastructure (VPC, networking, databases) and Elastic Beanstalk to deploy applications on top. Or use Systems Manager for operational tasks like patching and configuration alongside CloudFormation.
CloudFormation Template Example (YAML)
AWSTemplateFormatVersion: '2010-09-09' Description: Simple EC2 Instance with Security Group Parameters: KeyName: Description: EC2 Key Pair for SSH access Type: AWS::EC2::KeyPair::KeyName Resources: WebServerSecurityGroup: Type: AWS::EC2::SecurityGroup Properties: GroupDescription: Allow HTTP and SSH SecurityGroupIngress: - IpProtocol: tcp FromPort: 80 ToPort: 80 CidrIp: 0.0.0.0/0 - IpProtocol: tcp FromPort: 22 ToPort: 22 CidrIp: 0.0.0.0/0 WebServerInstance: Type: AWS::EC2::Instance Properties: InstanceType: t2.micro ImageId: ami-0c55b159cbfafe1f0 # Amazon Linux 2 KeyName: !Ref KeyName SecurityGroups: - !Ref WebServerSecurityGroup UserData: Fn::Base64: !Sub | #!/bin/bash yum update -y yum install -y httpd systemctl start httpd systemctl enable httpd echo "Hello from CloudFormation!" > /var/www/html/index.html Outputs: PublicIP: Description: Public IP of web server Value: !GetAtt WebServerInstance.PublicIp WebsiteURL: Description: URL of website Value: !Sub 'http://${WebServerInstance.PublicDnsName}'
This CloudFormation template creates an EC2 instance with a web server. It defines a security group allowing HTTP and SSH access, launches an instance with Amazon Linux 2, and installs Apache web server using UserData. The template uses parameters for the SSH key and outputs the public IP and URL.
Elastic Beanstalk Deployment (CLI)
# Initialize Elastic Beanstalk application eb init -p python-3.8 my-web-app --region us-east-1 # Create an environment and deploy eb create production-env --instance-type t2.small # Deploy new application version eb deploy # Check environment status eb status # View application logs eb logs # Scale environment eb scale 3 # Terminate environment eb terminate production-env
These Elastic Beanstalk CLI commands show the typical workflow: initialize an application, create an environment, deploy code, monitor status, view logs, scale the application, and terminate when done. Elastic Beanstalk handles all the underlying infrastructure automatically.
Elastic Beanstalk Configuration (.ebextensions)
# .ebextensions/environment.config option_settings: aws:elasticbeanstalk:application:environment: DATABASE_URL: "mysql://username:password@host:3306/dbname" SECRET_KEY: "your-secret-key" aws:autoscaling:launchconfiguration: InstanceType: t2.small aws:autoscaling:asg: MinSize: 2 MaxSize: 6 aws:elasticbeanstalk:environment: LoadBalancerType: application aws:elasticbeanstalk:healthreporting:system: SystemType: enhanced packages: yum: postgresql-devel: [] commands: 01_install_dependencies: command: "pip install -r requirements.txt"
This .ebextensions configuration file customizes an Elastic Beanstalk environment. It sets environment variables, configures auto-scaling (min 2, max 6 instances), specifies instance type, enables Application Load Balancer, activates enhanced health reporting, and installs additional packages and dependencies.
Systems Manager - Parameter Store (CLI)
# Store a parameter aws ssm put-parameter \ --name "/myapp/database/password" \ --value "MySecurePassword123" \ --type "SecureString" \ --description "Database password for myapp" # Retrieve a parameter aws ssm get-parameter \ --name "/myapp/database/password" \ --with-decryption # List all parameters aws ssm describe-parameters # Delete a parameter aws ssm delete-parameter \ --name "/myapp/database/password" # Batch get parameters aws ssm get-parameters \ --names "/myapp/database/host" "/myapp/database/port" "/myapp/database/password" \ --with-decryption
AWS Systems Manager Parameter Store provides secure, hierarchical storage for configuration data and secrets. These CLI commands demonstrate creating encrypted parameters, retrieving them with decryption, listing all parameters, and batch operations. Parameters can be referenced in CloudFormation, Lambda, and other services.

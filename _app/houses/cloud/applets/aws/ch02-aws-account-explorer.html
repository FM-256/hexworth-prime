üë§ Root User Account
‚ñº

The root user has complete access to all AWS services and resources in your account. It's created when you first sign up for AWS and should be protected with the highest security measures.

Email Address Login
Uses the email address provided during account creation
Unrestricted Access
Has permissions to all resources and cannot be restricted
Billing Access
Only account type that can view and manage billing by default
Critical Tasks Only
Should only be used for tasks that require root access
üîí Security Best Practices
Always: Enable MFA, use a strong password, never share credentials
Never: Use root for everyday tasks, create access keys for root user
Instead: Create IAM users with appropriate permissions for daily operations
üîê IAM (Identity and Access Management)
‚ñº

IAM enables you to manage access to AWS services and resources securely. Create and manage users, groups, and roles with specific permissions.

IAM Users
Individual identities with unique credentials for people or applications
IAM Groups
Collections of users with shared permissions for easier management
IAM Roles
Temporary credentials for services, applications, or federated users
IAM Policies
JSON documents that define permissions for users, groups, and roles

üí∞ AWS Pricing Model
AWS follows a pay-as-you-go pricing model. You only pay for the resources you use, with no upfront costs or long-term commitments (unless you choose reserved instances for additional savings).

Scenario-Based Recommendations

Traditional Web Application with Database
You need to host a traditional web application (PHP, Python, Ruby) with a relational database. You want minimal configuration and automatic scaling.
Recommended: Elastic Beanstalk
  • Automatic load balancing and auto-scaling
  • Easy database integration with RDS
  • Built-in monitoring and health checks
  • Simple deployment via ZIP file or Git
Event-Driven Data Processing
Process files uploaded to S3, resize images, transform data, or respond to API Gateway requests. Workload is sporadic with unpredictable traffic patterns.
Recommended: AWS Lambda
  • Pay only for execution time (no idle costs)
  • Automatic scaling from zero to thousands
  • No server management required
  • Integrated with 200+ AWS services
Microservices Architecture with Containers
Run multiple containerized microservices that need to scale independently. You have Docker expertise and want orchestration capabilities.
Recommended: Amazon ECS
  • Native Docker support with AWS integration
  • Service discovery and load balancing
  • Can run on Fargate (serverless) or EC2
  • Integrated with CloudWatch for monitoring
High-Performance Computing or Custom Infrastructure
You need complete control over the operating system, network configuration, and installed software. Running specialized workloads like HPC, gaming servers, or custom applications.
Recommended: Amazon EC2
  • Full root access and OS-level control
  • Choice of processors, storage, and networking
  • Support for GPU instances and specialized hardware
  • Persistent storage with EBS volumes
Machine Learning Model Inference
Deploy machine learning models for real-time predictions with automatic scaling. Traffic patterns are unpredictable and can spike suddenly.
Recommended: AWS Lambda or ECS on Fargate
  • Lambda: For lightweight models under 15 minutes execution
  • ECS: For larger models requiring more memory/GPU
  • Both support auto-scaling based on demand
  • Integration with SageMaker for ML workflows

Detailed Service Breakdown

Amazon EC2 - Virtual Servers

1
Choose AMI (Amazon Machine Image)
Select your operating system: Amazon Linux, Ubuntu, Windows Server, etc.
2
Select Instance Type
Pick CPU, memory, and network capacity: t2.micro, m5.large, c5.xlarge, etc.
3
Configure Storage
Attach EBS volumes for persistent storage with customizable size and performance.
4
Set Security Groups
Define firewall rules for inbound and outbound traffic.
5
Launch & Connect
Start the instance and connect via SSH (Linux) or RDP (Windows).

Key Features:

  • Auto Scaling: Automatically adjust capacity based on demand
  • Load Balancing: Distribute traffic across multiple instances
  • Spot Instances: Up to 90% discount for interruptible workloads
  • Reserved Instances: Save up to 75% with 1-3 year commitments
  • Elastic IPs: Static IP addresses that can be reassigned

Pricing Example:

Instance Type vCPU Memory Price (On-Demand)
t2.micro 1 1 GB $0.0116/hour
t3.medium 2 4 GB $0.0416/hour
m5.large 2 8 GB $0.096/hour

AWS Lambda - Serverless Functions

1
Write Function Code
Develop in Python, Node.js, Java, Go, .NET, or Ruby.
2
Configure Trigger
Define event source: S3, API Gateway, CloudWatch Events, DynamoDB, etc.
3
Set Memory & Timeout
Allocate 128 MB to 10 GB memory, up to 15 minutes execution time.
4
Deploy & Monitor
Lambda automatically scales and logs to CloudWatch.

Example Function:

import json import boto3 def lambda_handler(event, context): # Process S3 event s3 = boto3.client('s3') bucket = event['Records'][0]['s3']['bucket']['name'] key = event['Records'][0]['s3']['object']['key'] # Get object response = s3.get_object(Bucket=bucket, Key=key) # Process data content = response['Body'].read().decode('utf-8') return { 'statusCode': 200, 'body': json.dumps('Processing complete!') }

Pricing:

  • Free Tier: 1 million requests + 400,000 GB-seconds per month
  • Requests: $0.20 per 1 million requests
  • Duration: $0.0000166667 per GB-second
  • Example: 1 million executions at 512 MB for 1 second = $3.33/month

Amazon ECS - Container Orchestration

Two Launch Types:

Fargate (Serverless)

  • No server management
  • Pay per task
  • Automatic scaling
  • Ideal for most workloads

EC2 Launch Type

  • Run on your EC2 instances
  • More control over infrastructure
  • Better for spot instances
  • Reserved Instance discounts

Task Definition Example:

{ "family": "web-app", "containerDefinitions": [ { "name": "nginx", "image": "nginx:latest", "cpu": 256, "memory": 512, "portMappings": [ { "containerPort": 80, "protocol": "tcp" } ], "environment": [ { "name": "ENV", "value": "production" } ] } ] }

What Beanstalk Manages:

  • Capacity Provisioning: Automatically provisions EC2 instances
  • Load Balancing: Sets up and configures ELB
  • Auto Scaling: Configures scaling policies
  • Health Monitoring: CloudWatch integration and dashboards
  • Platform Updates: Managed patching and updates
  • Deployment: Rolling, blue/green, or immutable deployments

Deployment:

# Install EB CLI pip install awsebcli # Initialize Beanstalk application eb init -p python-3.8 my-app # Create environment and deploy eb create my-env eb deploy # Open application in browser eb open

VPC (Virtual Private Cloud)
A logically isolated virtual network in the AWS cloud where you can launch AWS resources in a defined network space.
  • Define your own IP address range (CIDR block)
  • Create subnets in different Availability Zones
  • Configure route tables and network gateways
  • Multiple layers of security (security groups & NACLs)
  • Default VPC created automatically in each region
  • Can create custom VPCs with custom configurations
  • VPC peering allows connection between VPCs
  • PrivateLink for private access to services
üì¶
Subnets
A range of IP addresses in your VPC. Subnets reside in a single Availability Zone and can be public or private.
  • Public Subnet: Has route to Internet Gateway, resources get public IPs
  • Private Subnet: No direct internet access, uses NAT for outbound
  • Each subnet resides in one Availability Zone
  • Subnet CIDR block must be subset of VPC CIDR
  • AWS reserves 5 IP addresses in each subnet
  • Can have multiple subnets per AZ
  • Use different subnets for different tiers (web, app, database)
  • Network ACLs control traffic at subnet level

Route Tables
Contains rules (routes) that determine where network traffic from your subnet or gateway is directed.
  • Each VPC has a main route table
  • Can create custom route tables for specific subnets
  • Routes specify destination CIDR and target
  • Local route enables communication within VPC
  • 0.0.0.0/0 route points to Internet Gateway for internet access
  • Can route to NAT Gateway, VPN, VPC Peering
  • Most specific route takes precedence
  • Subnet associations determine which route table is used
üåê
Internet Gateway (IGW)
A horizontally scaled, redundant, and highly available VPC component that allows communication between instances in your VPC and the internet.
  • Provides a target in route tables for internet-routable traffic
  • Performs NAT for instances with public IP addresses
  • Only one IGW can be attached to a VPC
  • No availability risks or bandwidth constraints
  • Supports both IPv4 and IPv6
  • No additional charges for IGW usage
  • Must update route table to enable internet access
  • Resources need public IP or Elastic IP
üîÑ
NAT Gateway
Enables instances in a private subnet to connect to the internet or other AWS services while preventing the internet from initiating connections.
  • Managed service with built-in redundancy
  • Must be created in a public subnet
  • Requires an Elastic IP address
  • Private subnet route table points to NAT Gateway
  • Supports up to 45 Gbps bandwidth
  • Charged per hour and per GB processed
  • Deploy one NAT Gateway per AZ for HA
  • Alternative: NAT Instance (self-managed EC2)

Security Groups
Virtual firewall that controls inbound and outbound traffic for AWS resources (instance-level security).
  • Stateful: Return traffic automatically allowed
  • Allow rules only (no explicit deny)
  • Operates at the instance level
  • Can reference other security groups
  • Default: All outbound allowed, all inbound denied
  • Changes take effect immediately
  • Can assign up to 5 security groups per instance
  • Best practice: Separate SGs for web, app, DB tiers
üöß
Network ACLs
Optional layer of security that acts as a firewall for controlling traffic in and out of subnets (subnet-level security).
  • Stateless: Must allow both inbound and outbound
  • Allow and deny rules supported
  • Operates at the subnet level
  • Rules processed in number order (lowest first)
  • Default NACL allows all traffic
  • Custom NACLs deny all traffic by default
  • Separate inbound and outbound rules
  • Use for additional subnet-level security

Elastic Load Balancer
Automatically distributes incoming application traffic across multiple targets in multiple Availability Zones.
  • Application Load Balancer (ALB): Layer 7, HTTP/HTTPS traffic
  • Network Load Balancer (NLB): Layer 4, ultra-low latency
  • Gateway Load Balancer (GWLB): Layer 3, virtual appliances
  • Classic Load Balancer (CLB): Legacy, Layer 4/7
  • Health checks ensure traffic only to healthy targets
  • Automatic scaling to handle traffic load
  • SSL/TLS termination capabilities
  • Cross-zone load balancing distributes evenly

An AWS Region is a physical location around the world where AWS clusters data centers. Each region consists of multiple isolated Availability Zones. Regions are completely independent, allowing for geographic redundancy and data residency compliance.

The Internet Gateway is attached to the VPC and provides a target for internet-routable traffic. Public subnets have a route (0.0.0.0/0) pointing to the IGW, enabling instances with public IPs to communicate with the internet.

Private Subnet (10.0.3.0/24): Contains application servers with no public IPs, routes through NAT Gateway for outbound internet access.

Private Subnet (10.0.4.0/24): Contains database servers, isolated from internet, communicates only with app tier through security groups.

The ALB sits in both public subnets (AZ-A and AZ-B) and distributes incoming HTTP/HTTPS traffic across web servers in both zones. It performs health checks and only routes to healthy instances. Provides SSL termination and path-based routing.

üîß VPC Components Deep Dive

Amazon VPC lets you provision a logically isolated section of the AWS Cloud. Understanding each component is crucial for designing secure and scalable architectures.

VPC Endpoints
Enable private connections between VPC and AWS services without requiring internet gateway, NAT, VPN, or AWS Direct Connect. Two types: Interface Endpoints (powered by PrivateLink) and Gateway Endpoints (S3 and DynamoDB).
VPC Peering
Network connection between two VPCs that enables routing traffic between them using private IP addresses. Works across regions and accounts. Not transitive - requires direct peering relationship.
Transit Gateway
Acts as a cloud router to connect VPCs and on-premises networks through a central hub. Simplifies network topology and reduces the number of connections needed.
VPN Connection
Secure connection between on-premises network and AWS VPC over the internet. Uses IPsec for encryption. Consists of Virtual Private Gateway (AWS side) and Customer Gateway (on-prem side).
Direct Connect
Dedicated network connection from on-premises to AWS. Provides more consistent network performance than internet-based connections. Useful for large data transfers and hybrid architectures.
Elastic IP
Static IPv4 address designed for dynamic cloud computing. Associated with your account, not a specific instance. Can be quickly remapped to another instance in case of failure.
Flow Logs
Captures information about IP traffic going to and from network interfaces in VPC. Useful for monitoring, troubleshooting, and security analysis. Can be published to CloudWatch Logs or S3.
PrivateLink
Provides private connectivity between VPCs, AWS services, and on-premises applications without exposing traffic to the internet. More secure than VPC peering for service access.
üéØ Best Practices for VPC Design
Plan IP Address Space
Choose CIDR blocks that won't overlap with other networks (on-prem, other VPCs). Use /16 for VPC (65,536 addresses), /24 for subnets (256 addresses). Leave room for growth.
Multi-AZ Deployment
Distribute resources across at least 2 Availability Zones for high availability. Create subnets in multiple AZs. Deploy NAT Gateways in each AZ for fault tolerance.
Use Private Subnets
Place application and database tiers in private subnets without direct internet access. Only expose what's necessary (web tier) in public subnets. Use bastion hosts for admin access.
Defense in Depth
Use both security groups and NACLs. Security groups for instance-level control, NACLs for subnet-level control. Implement least privilege access principles.
Monitor with Flow Logs
Enable VPC Flow Logs for security analysis and troubleshooting. Monitor for unusual traffic patterns. Integrate with CloudWatch and security tools for automated alerts.
Use VPC Endpoints
Access AWS services privately without NAT Gateway costs. Improves security by keeping traffic on AWS network. Supports S3, DynamoDB, and many other services via PrivateLink.

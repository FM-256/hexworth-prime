üîê Biometric Devices

Biometric authentication uses unique physical characteristics to verify identity. Provides "something you are" authentication factor.

üëÜ Fingerprint

Most common biometric

  • Optical (camera-based)
  • Capacitive (electrical)
  • Ultrasonic (sound waves)

üëÅÔ∏è Facial Recognition

Face geometry mapping

  • 2D camera (less secure)
  • 3D depth sensing
  • IR for low light

üëÅÔ∏è Iris/Retina

Eye pattern scanning

  • Iris: Pattern of colored ring
  • Retina: Blood vessel pattern
  • Highly accurate

Biometric Considerations

Factor Description
False Acceptance Rate (FAR) Incorrectly accepting unauthorized user (security risk)
False Rejection Rate (FRR) Incorrectly rejecting authorized user (usability issue)
Crossover Error Rate (CER) Point where FAR equals FRR - lower is better
Throughput How quickly users can authenticate
üîí Security Note: Biometrics can't be changed if compromised (unlike passwords). Use multi-factor authentication combining biometrics with passwords or tokens.

üîç Barcode & QR Scanners

Scanner Types

Type Technology
Laser Red laser beam, 1D barcodes only
CCD (LED) LED array, close range, 1D codes
2D Imager Camera-based, reads QR codes & 2D
Omni-directional Multiple scan lines, any orientation

Connection Types

  • USB (HID): Acts as keyboard, types scanned data
  • USB (COM): Serial communication, needs driver
  • Bluetooth: Wireless, paired to device
  • Wedge: Inline with keyboard (legacy)
üìä 1D vs 2D: Traditional barcodes are 1D (lines). QR codes are 2D (matrix) and store much more data including URLs, text, and contact info.

üì† Document Scanners

Type Best For

Key Specifications

  • Resolution (DPI): Higher = more detail (300 DPI for documents, 600+ for photos)
  • Color Depth: Bits per pixel (24-bit = 16.7M colors)
  • ADF Capacity: Pages auto document feeder can hold
  • Duplex: Scans both sides automatically
üí° OCR (Optical Character Recognition): Software that converts scanned images to editable text. Quality depends on scan resolution, document condition, and font clarity.

üéÆ Game Controllers & Motion Sensors

Controller Types

  • Gamepad: Console-style controllers (Xbox, PlayStation)
  • Joystick: Flight simulators, precise control
  • Racing Wheel: Force feedback, pedals
  • Flight Stick (HOTAS): Hands-on throttle and stick
  • Arcade Stick: Fighting games, large buttons

Motion & VR Input

  • Accelerometer: Detects tilt and motion
  • Gyroscope: Detects rotation
  • Depth Camera: Tracks body position (Kinect)
  • VR Controllers: 6DOF tracking, haptic feedback
üéØ XInput vs DirectInput: XInput is the modern Windows controller API (Xbox controllers). DirectInput is legacy but supports more buttons/axes. Many games support both.

üñ•Ô∏è KVM Switches

KVM (Keyboard, Video, Mouse) switches allow one set of peripherals to control multiple computers.

Types

  • Hardware KVM: Physical switch with cables
  • Software KVM: Network-based (Synergy, Mouse Without Borders)
  • KVM over IP: Remote access over network

Features to Consider

  • Number of ports (2, 4, 8, 16+)
  • Video support (VGA, DVI, HDMI, DP)
  • USB passthrough for other devices
  • Hotkey switching vs button
  • Audio support
üíº Use Case: IT administrators managing multiple servers, developers switching between test machines, or home users with multiple PCs at one desk.

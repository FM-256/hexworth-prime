Windows Administrative Tools

What Are Administrative Tools?

Administrative Tools (also called Windows Tools in Windows 11) is a collection of advanced system management utilities. These are the professional-grade tools that IT administrators use to manage, monitor, troubleshoot, and configure Windows systems.

üéØ

Why They Matter

These tools go far beyond what Settings or Control Panel offer. They provide deep system access for tasks like managing services, viewing system logs, scheduling automated tasks, configuring security policies, and troubleshooting hardware issues.

You'll learn to navigate the MMC framework, use Computer Management as your Swiss Army knife, manage devices and disks, interpret Event Viewer logs, control services, schedule tasks, configure security policies, and create custom management consoles.

  • Most tools are based on the MMC (Microsoft Management Console) framework

Microsoft Management Console (MMC)

The framework behind Administrative Tools

üéØ Understanding MMC

MMC is NOT an administrative tool itself. It's a framework - a container that hosts administrative tools called "snap-ins."

Think of MMC as a picture frame, and snap-ins as the pictures you put in it. Computer Management, Device Manager, Services, Event Viewer - these are all MMC snap-ins running inside the MMC framework.

Snap-ins are modular administrative tools that plug into MMC. Examples: Device Manager, Disk Management, Services, Event Viewer, Performance Monitor. You can combine multiple snap-ins into a single custom console.

Disk Management

Partition and volume configuration

üíæ What Is Disk Management?

diskmgmt.msc

Disk Management allows you to create, format, resize, and delete partitions without third-party tools. It's built into Windows and provides both graphical and list views of all storage devices.

üîß Core Capabilities

  • Initialize new disks: MBR or GPT partition style
  • Create volumes: Simple, spanned, striped, mirrored, RAID-5
  • Format partitions: NTFS, FAT32, exFAT, ReFS
  • Assign drive letters: Change or remove letters
  • Extend/shrink volumes: Resize existing partitions
  • Mark partition active: Set boot partition
üìä Understanding Disk Management Views

Upper pane (Volume List): All volumes listed with details (filesystem, capacity, status, device type)

Lower pane (Graphical View): Visual representation of physical disks and their partitions. Color-coded by status and type.

Color codes: Blue = Primary partition | Green = Logical drive | Purple/Magenta = Dynamic volumes | Black stripe = Unallocated space

üÜï

Initialize a New Disk

1. Connect new drive
2. Disk Management auto-prompts initialization
3. Choose partition style:
   MBR: Up to 2TB, max 4 primary partitions
   GPT: >2TB, unlimited partitions, modern standard
4. Right-click unallocated ‚Üí New Simple Volume

üìè

Extend/Shrink Volume

Shrink: Right-click volume ‚Üí Shrink Volume ‚Üí Enter size
(Creates unallocated space from existing partition)

Extend: Right-click volume ‚Üí Extend Volume ‚Üí Use unallocated space
(Limitation: Unallocated space must be adjacent and to the right)

üî§

Change Drive Letter

1. Right-click volume ‚Üí Change Drive Letter and Paths
2. Click Change
3. Select new letter from dropdown
4. Click OK (no reboot needed)

Pro tip: System/Boot drives cannot be changed

DISK MANAGEMENT SAFETY RULES

  • Formatting ERASES ALL DATA - Double-check before confirming
  • Cannot extend C: drive to the left - Only adjacent right-side unallocated space works
  • Deleting a volume ‚â† Shrinking - Delete removes entire partition, shrink reduces size
  • Always backup before disk operations - Power loss during resize = data loss risk

Event Viewer

The system's black box recorder

üìä Why Event Viewer Matters

Event Viewer (eventvwr.msc) logs EVERYTHING that happens on your system: successful logins, failed logins, service crashes, driver installations, security alerts, application errors, and system warnings.

If something went wrong, Event Viewer knows about it.

üìÅ Windows Logs

Primary log categories
  • Application: Software events (crashes, errors, warnings)
  • Security: Login attempts, privilege use, audit events
  • Setup: Windows Update and installation events
  • System: Drivers, services, hardware, core OS events
  • Forwarded Events: Logs from other computers (enterprise)

üéØ Event Levels

  • Information (blue i): Normal operation events
  • Warning (yellow !): Potential problems, not critical
  • Error (red X): Significant problem, function failure
  • Critical (red circle): System instability or data loss
  • Success Audit (key): Successful security event
  • Failure Audit (key+X): Failed security event

üîç Applications and Services Logs

More detailed, component-specific logs:
‚Ä¢ Microsoft ‚Üí Windows ‚Üí (specific component)
‚Ä¢ Example: DNS-Client, NTFS, PowerShell, TaskScheduler

These logs provide granular troubleshooting data for specific Windows features.

üõ†Ô∏è Real-World Event Viewer Workflow

Scenario: Computer crashes randomly. How do you diagnose?

  1. Open Event Viewer ‚Üí Windows Logs ‚Üí System
  2. Filter by: Error and Critical levels only
  3. Look for events timestamped near crash time
  4. Common culprits: Event ID 41 (unexpected shutdown), Event ID 1001 (bugcheck/BSOD)
  5. Double-click event ‚Üí Details ‚Üí Copy event data ‚Üí Google search
  6. Cross-reference with Application log for software-related crashes

EVENT VIEWER PRO TIPS

  • Create Custom Views: Filter specific Event IDs across multiple logs
  • Event ID is key: Search "Windows Event ID [number]" for solutions
  • Security log requires auditing enabled: Group Policy ‚Üí Audit Policy
  • Export logs: Save Event Log ‚Üí Share with support/analyze offline

Services

Windows background process management

‚öôÔ∏è What Are Services?

Services are background programs that run without user interaction. They start before login, continue after logout, and power critical Windows functionality: networking, security, printing, updates, Bluetooth, audio, and hundreds more.

Services = The engine room of Windows. Most users never see them. Admins live here.

üéõÔ∏è Services Management

services.msc
  • Start/Stop/Restart: Control running state
  • Startup Type: Automatic, Manual, Disabled
  • Recovery: Actions on failure (restart, run program)
  • Dependencies: View/manage service requirements
  • Log On: Account that runs the service

üö¶ Startup Types Explained

  • Automatic: Starts at boot (traditional)
  • Automatic (Delayed): Starts after boot completes (reduces startup time)
  • Manual: Only starts when needed by another service/program
  • Disabled: Cannot start (use for unnecessary/problematic services)

üîÑ Service Status

Running: Service is active
Stopped: Service is not running
Paused: Service temporarily halted (rare)
(blank): Manual service, not started

Status ‚â† Startup Type - A service can be set to Automatic but show Stopped status if it failed to start.

üõ†Ô∏è Common Service Troubleshooting

Problem: Print Spooler keeps stopping

  1. Open services.msc ‚Üí Find "Print Spooler"
  2. Double-click ‚Üí Recovery tab
  3. First failure: Restart the Service
  4. Second failure: Restart the Service
  5. Subsequent failures: Restart the Service
  6. Reset fail count after: 1 day

Now the service auto-recovers instead of requiring manual intervention.

CRITICAL SERVICE KNOWLEDGE

  • Dependencies matter: Stopping a parent service stops dependent children
  • Check Event Viewer if service won't start: System log shows failure reasons
  • Don't randomly disable services: Can break Windows functionality
  • Use delayed start for performance: Reduces boot time congestion

Task Scheduler & Security Policies

Automation and security configuration

‚è∞ Task Scheduler

taskschd.msc

Automate program execution based on triggers: time, events, login, idle, workstation lock/unlock, and more.

  • Triggers: Schedule, event, login, startup, idle
  • Actions: Run program, send email, display message
  • Conditions: AC power, idle time, network connection
  • Settings: Retry behavior, stop if runs too long
  • History: Track execution success/failure

üìã Creating a Basic Task

  • Action ‚Üí Create Basic Task
  • Name and description
  • Trigger: When do you want to run?
  • Action: What should it do?
  • Summary and finish
  • Advanced: Create Task (more options, elevated privileges)
üõ°Ô∏è Example: Schedule a PowerShell Script

Goal: Run a backup script every night at 2 AM with admin privileges

Setup:

  1. Create Task (not Basic Task - need elevated options)
  2. General tab: Check "Run with highest privileges"
  3. Triggers tab: New ‚Üí Daily ‚Üí 2:00 AM
  4. Actions tab: New ‚Üí Start a program ‚Üí powershell.exe -File "C:\Scripts\backup.ps1"
  5. Conditions tab: Uncheck "Start only if on AC power" (for laptops)
  6. Settings tab: Check "Run task as soon as possible after scheduled start is missed"

üîê Local Security Policy

secpol.msc

Available on Pro/Enterprise only

  • Password policies (complexity, length, age)
  • Account lockout policies (failed login attempts)
  • User rights assignment (who can do what)
  • Security options (UAC, network access)
  • Audit policies (what gets logged)

‚öôÔ∏è Group Policy Editor

gpedit.msc

Pro/Enterprise only - Home edition excluded

  • Local Computer Policy configuration
  • User and Computer settings
  • Windows Update control
  • Restrict access to Control Panel items
  • Script execution policies
  • Much more powerful than Security Policy

IMPORTANT DISTINCTIONS

  • Task Scheduler: All editions - Schedule automation
  • Local Security Policy: Pro/Enterprise - Security-specific policies
  • Group Policy Editor: Pro/Enterprise - Comprehensive system policies
  • Home edition users: Can modify some policies via registry (advanced)

Creating Custom MMC Consoles

Build your own administrative toolbox

üéØ Why Create Custom Consoles?

Instead of opening 5 different tools, create ONE console with all your frequently-used snap-ins. Perfect for:

  • Role-specific toolsets (Network Admin, Security Admin, Help Desk)
  • Streamlined workflows (combine related tools)
  • Training environments (provide students with pre-configured tools)
  • Quick troubleshooting (all diagnostic tools in one place)
üõ†Ô∏è Step-by-Step: Build a Network Admin Console

Goal: Create a console with Event Viewer, Services, Device Manager, and Performance Monitor

  1. Launch blank MMC: Win + R ‚Üí mmc
  2. Add snap-ins: File ‚Üí Add/Remove Snap-ins (Ctrl+M)
  3. Select "Event Viewer" ‚Üí Add ‚Üí OK
  4. Select "Services" ‚Üí Add ‚Üí Local computer ‚Üí OK
  5. Select "Device Manager" ‚Üí Add ‚Üí Local computer ‚Üí OK
  6. Select "Performance Monitor" ‚Üí Add ‚Üí OK
  7. Click OK to close snap-in window
  8. Save console: File ‚Üí Save As ‚Üí NetworkAdmin.msc
  9. Save to Desktop or Documents for easy access
  10. Future use: Double-click NetworkAdmin.msc ‚Üí All tools load instantly
üîß

Console Modes

When saving, choose console mode:

Author mode: Full access, can modify console
User mode - full access: Can't modify console, full snap-in access
User mode - limited: Restricted to specific tools
User mode - single window: No navigation, specific view only

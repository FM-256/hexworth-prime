Post-Incident

üìã Phase 1: Preparation

  • Develop and maintain IR policies and procedures
  • Build and train the incident response team (IRT)
  • Deploy security tools and monitoring infrastructure
  • Establish communication plans (internal & external)
  • Create contact lists for stakeholders and third parties
  • Set up evidence collection and preservation procedures
  • Conduct tabletop exercises and simulations
  • Maintain up-to-date asset inventory

üîç Phase 2: Identification

  • Detection: Monitor alerts from SIEM, IDS/IPS, EDR
  • Analysis: Triage and validate potential incidents
  • Collection: Gather initial evidence and logs
  • Determine incident scope, severity, and impact
  • Identify affected systems, users, and data
  • Document timeline of events
  • Assign incident classification and priority
  • Notify appropriate stakeholders

üõ°Ô∏è Phase 3: Containment

  • Short-term: Isolate affected systems immediately
  • Long-term: Apply temporary fixes and patches
  • Block malicious IPs, domains, and hashes
  • Disable compromised accounts
  • Implement network segmentation
  • Preserve evidence before changes
  • Deploy additional monitoring on affected segments
  • Document all containment actions taken

üßπ Phase 4: Eradication

  • Remove malware and malicious artifacts
  • Delete unauthorized accounts and access
  • Apply permanent patches and fixes
  • Reset compromised credentials
  • Clean or reimage affected systems
  • Update firewall and security rules
  • Verify complete removal of threat
  • Address root cause vulnerabilities

üîß Phase 5: Recovery

  • Restore systems from clean backups
  • Rebuild systems if necessary
  • Verify system integrity before production
  • Monitor closely for signs of reinfection
  • Gradually restore services and access
  • Validate business processes are functioning
  • Confirm security controls are operational
  • Document recovery timeline

üìä Phase 6: Post-Incident

  • Root Cause Analysis: Identify how and why incident occurred
  • Lessons Learned: Conduct after-action review meeting
  • Documentation: Complete incident report including:
    • Executive Summary
    • Incident Description & Timeline
    • Technical Analysis
    • Impact Assessment
    • Remediation Actions
    • Recommendations
  • Update IR procedures based on findings
  • Improve detection and prevention controls
  • Share threat intelligence (if appropriate)

Network-Based Sources

Source Type Key Information

IDS/IPS Logs Network Attack signatures, alerts, blocked threats
WAF Logs Network Web attacks, SQLi, XSS attempts

NetFlow/IPFIX Network Traffic patterns, volume, flow data
DNS Logs Network Domain queries, potential C2 domains

Source Type Key Information

Syslog Host Linux system events, service activity
Registry Host Persistence, configuration changes

File System Host Artifacts, timestamps, suspicious files
EDR Telemetry Host Process trees, file changes, network activity

Source Type Key Information

What Should You Log?

Critical data for security monitoring

📋
Events
What is taking place with resources and infrastructure? Identify potential malicious or anomalous events.
👤
Activity
What is being done with resources? Identify insider behavior that violates policies.
📊
Metrics
How are resources performing? Determine if resources are acting differently than expected.

📊 SIEM vs SOAR

SIEM (Security Information & Event Management)

  • Streamlines security efforts
  • Aggregates info into single source
  • Security incident analysis & investigation
  • Cloud SIEM via single platform
  • Examples: Microsoft Sentinel, Chronicle

SOAR (Security Orchestration & Automated Response)

  • Works with SIEM for efficiency
  • Uses playbooks and automation
  • Proactive threat response
  • Minimal human interaction needed
  • Auto-remediate common threats

🛡 Cloud Security Posture Management

CSPM

  • Automated software for identifying security risks
  • Inspects SaaS, PaaS, IaaS, containers, serverless
  • Runs in background analyzing compliance risks
  • Scans multi-cloud environments
  • Detects configuration vulnerabilities

CNAPP (Cloud-Native Application Protection)

  • Combines CSPM, CWPP, and CIEM
  • Full coverage and visibility
  • Shift-left capabilities
  • Holistic view of attack vectors
  • Context-aware risk analysis

📝 Incident Response Planning

Have a documented Incident Response Plan that defines: incidents by service/infrastructure, roles and responsibilities, detection to resolution process, communication plan (including media if needed), and legal/regulatory notification requirements.

💡 Key Takeaways

✓ Log Events, Activity, and Metrics for security monitoring
✓ SIEM aggregates security data; SOAR automates response
✓ CSPM identifies cloud security risks and misconfigurations
✓ CNAPP combines CSPM + CWPP + CIEM for full protection
✓ Prioritize incidents by Impact x Urgency
✓ Document IR plan with communication and notification requirements
